ExponentiationBySquaring



Modulo:
What is the value of ( 3 ^ 20 ) % 7?

According to multiplication rule of modular arithmetic,

( a * b * c ) % c == ( ( a % m ) * ( b % m ) * ( c % m  ) ) % m

So, ( 3 ^ 20 ) % 7 ==( ( 3 ^ 10 ) * ( 3 ^ 10 ) ) % 7 == ( ( 3 ^ 10 ) % 7 * ( 3 ^ 10 ) % 7 ) % 7.

1. Exponentiation by squaring: Method for calculation the value of x ^ n in quickest way !

Example: 

0.

35 =((3)2)2×3 = 92×3 = 243    _ only 3 time multiplication instead of 5 times multiplication

1. The formula a7×b5 may be calculated within 3 steps:

((a)2×a)2×a (four multiplications for calculating a7)
((b)2)2×b (three multiplications for calculating b5)
(a7)×(b5) (one multiplication to calculate the product of the two)

so one gets eight multiplications in total.

A faster solution is to calculate both powers simultaneously:

((a×b)2×a)2×a×b

which needs only 6 multiplications in total. Note that a×b is calculated twice, the result could be stored after the first calculation which reduces the count of multiplication to 5.

2. Example with numbers:

27×35 = ((2×3)2×2)2×2×3 = (62×2)2×6 = 722×6 = 31,104

   Calculating the powers simultaneously instead of calculating them separately always reduces the count of multiplications if at least two of the exponents are greater than 1.

C++ Code:

long long power(long long x, long long n) {
	long long result =1;
	while(n!=0){
		if(n%2){
			result *= x;
			n -= 1;
		}
		else {
			x *= x;
			n /= 2;
			}	
	}
		return result;
	}
 
int main() {
	long long res =0,x = 5,n = 25;	
	res = power(x,n);
	cout<< res<<endl; // 5 ^ 25 = 298023223876953125	
	return 0;
	}

Modular Exponentiation  [Ref:Primality Testing : Non-deterministic Algorithms  -  http://community.topcoder.com/tc?module=Static&d1=tutorials&d2=primalityTesting]

1. Big-Mod Algorithm:  Compute:  (ab)%c

How to efficiently compute (ab)%c ( where a,b,c are non-negative integers ). A straightforward algorithm is 

/* a function to compute (ab)%c */
int modulo(int a,int b,int c){
    // res is kept as long long because intermediate results might overflow in "int"
    long long res = 1; 
    for(int i=0;i<b;i++){
        res *= a;
        res %= c; // this step is valid because (a*b)%c = ((a%c)*(b%c))%c
    }
    return res%c;
}

However, as you can clearly see, this algorithm takes O(b) time and is not very useful in practice. We can do it in O( log(b) ) by using what is called as exponentiation by squaring. The idea is very simple:

   (a2)(b/2)                if b is even and b > 0
    ab = a*(a2)((b-1)/2)    if b is odd
    1                       if b = 0

This idea can be implemented very easily as shown below:

/* This function calculates (ab)%c */
int modulo(int a,int b,int c){
    long long x=1,y=a; // long long is taken to avoid overflow of intermediate results
    while(b > 0){
        if(b%2 == 1){
            x=(x*y)%c;
        }
        y = (y*y)%c; // squaring the base
        b /= 2;
    }
    return x%c;
}


2. Compute:  (a*b)%c

The above code could only work for a,b,c in the range of type "int" or the intermediate results will run out of the range of "long long". To write a function for numbers up to 10^18, we need to compute (a*b)%c when computing a*b directly can grow larger than what a long long can handle. We can use a similar idea to do that:

     (2*a)*(b/2)                           if b is even and b > 0
     a*b = a + (2*a)*((b-1)/2)      if b is odd
     0                                           if b = 0 
/* this function calculates (a*b)%c taking into account that a*b might overflow */
long long mulmod(long long a,long long b,long long c){
    long long x = 0,y=a%c;
    while(b > 0){
        if(b%2 == 1){
            x = (x+y)%c;
        }
        y = (y*2)%c;
        b /= 2;
    }
    return x%c;
}

Miller-Rabin Primality Test - Non-deterministic Algorithms - finding whether a number is prime or not ?

Basic Method of testing Prime or not:

bool isprime ( int N )
{
    if ( N == 0 || N == 1 ) return false;
    if ( N % 2 == 0 ) return false;
    
    int i;
    int sqrtn = sqrt ( N );
    for ( i = 3; i <= sqrtn; i += 2 )    // Note: step + 2 = for avoiding testing the even numbers
    {
        if ( N % i == 0 ) return false;
    }
    
    return true;
}


Efficient method of finding prime number or not : Combining the Fermat's Little theorem and Miller-Rabin therorem.

  1. Fermat's Little Theorem.

        if p is a prime number and 'a' is a positive integer less than p, then 

    ap = a ( mod p )
       or 
    a(p-1) = 1 ( mod p )
 
   If p is the number which we want to test for primality, then we could randomly choose 'a', such that a < p and then calculate (a(p-1))%p. 
   If the result is not 1, then by Fermat's Little Theorem p cannot be prime.

  2. Miller - Rabin Theorem
     If p is prime and x2 = 1 ( mod p ), then x = +1 or -1 ( mod p ). We could prove this as follows:
     x2        = 1 ( mod p )
     x2 - 1    = 0 ( mod p )
    (x-1)(x+1) = 0 ( mod p )

Algorithm:
Let p be the given number which we have to test for primality. First we rewrite p-1 as (2d)*s. Now we pick some 'a' in range [1,n-1] and then check 
whether as = 1 ( mod p ) or a(s*(2r))= -1 ( mod p ). If both of them fail, then p is definitely composite. Otherwise p is probably prime. We can choose another a and 
repeat the same test. We can stop after some fixed number of iterations and claim that either p is definitely composite, or it is probably prime.

/* Miller-Rabin primality test, iteration signifies the accuracy of the test */
bool Miller(long long p,int iteration){
    if(p<2){
        return false;
    }
    if(p!=2 && p%2==0){
        return false;
    }
    long long s=p-1;
    while(s%2==0){
        s/=2;
    }
    for(int i=0;i<iteration;i++){
        long long a=rand()%(p-1)+1,temp=s;
        long long mod=modulo(a,temp,p);
        while(temp!=p-1 && mod!=1 && mod!=p-1){
            mod=mulmod(mod,mod,p);
            temp *= 2;
        }
        if(mod!=p-1 && temp%2==0){
            return false;
        }
    }
    return true;
}

int main() {
	long long p = 29,iter = 3;	
	bool res = false;
	res = Miller(p,iter);
	cout<< res<<endl; // true = 1 = Prime Number
	return 0;
}

Sieve of Eratosthenes - Fastest way of Generate Prime Numbers series for given the upper limit:

Theorem:

 If a number P is prime, then all of its multiples k*P ( where k > 1 ) are not prime.

Algorithm:

#include <iostream>
#include <bitset>
#define SIZE 10000000
#define MAX (int)(SIZE-3)/2
using namespace std;
 
int primes[MAX+1];                                     //array that stores the primes up to sqrt(SIZE)
bitset<MAX+1> bset;                                    //auxiliary bitset used to make the sieve
 
void Sieve_GeneratePrimes()
{
	int i,j;
	for(i=0;i*i<=SIZE;i++)                             //we only have to get primes up to sqrt(SIZE)
		if(!bset.test(i))
			 for(j=i+1;(2*j+1)*(2*i+3)<=SIZE;j++)
				  bset.set(((2*j+1)*(2*i+3)-3)/2);     //setting all non-primes
	primes[0]=2;                                       //store the first prime (that is 2)
	for(i=1,j=0;j<MAX+1;j++)
	   if(!bset.test(j))
		  primes[i++]=2*j+3;                           //store the remaining odd primes    
}
 
int main() {
	Sieve_GeneratePrimes();
	for(int i=0; i<100; ++i)                           // Upper limit can be up to 10000000
		cout<< primes[i] << " ";
	return 0;
}
