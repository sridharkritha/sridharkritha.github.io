Functional Template



Template:
    1. Template is not supported "automatic type conversion".
    2. Template code will be deduced only for member functions that are called - which saves time and space.
    3. If a class template has static members, these are instantiated once for each type.
    4. In the old version(NOT for the latest version) of C++ compiler, you should put white space between the two closing template brackets. If you don't
        do this, you are using operator >>, which results in a syntax error.    Ex: Stack<Stack<int> > obj;  // stack of stack of ints
    5. A class template can be partially specialized and/or fully specialized. A function template can only be fully specialized because function templates
        can be overload we can get nearly the same effect via overloading that we could have got via partial specialization.
    6. Unlike class templates, function templates will not support default template arguments but support default functional argument.

Class Template:
    template<class T, int SIZE = 100> // default template argument
    class Array {
    private:
        T TheArray[SIZE];
    };

Functional Template: C++ function may have default arguments. The defaultness may only go from right to left, meaning, if nth 
argument is required to be default,(n+1)th must also be default, and so on till last argument of function.
    template<class T>
    void Print(T array[], int array_size, T filter = T()) { // T() is zero for built-in types
        for(int nIndex = 0; nIndex < array_size; ++nIndex) {
            if ( array[nIndex] != filter) cout << array[nIndex];
        }
    }

Template Argument Full Usage: Every argument present in the template declaration should be used in the function argument definition.

1. No-argument template functions:
    template<class T>
    T test(void)    // Error: T is not used as an argument
    {
        T temp;
    }

2. Usage of partial template arguments:
    template<class T, class U>
    void test(T &a)    // Error: U is not used as an argument
    {
        U temp;
    }

3. Usage of non template arguments:
    template<class T, int size>  // Error: non template arguments(int size) is not allowed in functional template 
    void test(T &a, int size) { // but allowed in class template
        T temp;
    }


A. Function Template:
Approach 1: 
    template <class T>
    inline T const& max (T const& a, T const& b) {  
        return a < b ? b : a;
    }

    int main() {
        double f1 = -6.7;
        max(f1,3.4); // OK 
    
        std::string s1 = "mathematics";
        std::string s2 = "math";
        max(s1,s2); // OK 

        max(42,66.66);                   // ERROR: first T is int, second T is double
        max(static_cast<double>(4),4.2); // OK - by explicit type conversion
        max<double>(4,4.2);              // OK - by explicit type conversion - first argument
// converts from int to double
    }

Approach 2: Re -define the template function for avoiding the explicit type conversion:
    template <typename T1, typename T2>
    inline T1 max (T1 const& a, T2 const& b) {
        return a < b ? b : a;
    }

    int main() {
        max(42,66.66) ; // OK but it has following drawbacks
    }

Drawbacks:
1. If the order of parameters change then the return value also change
    Ex: max(42,66.66)  => 66
        max(66.66,42)  => 66.66
2. For the function call max(42,66.66) the final return value integer 66 is internally converted from double 66.66 using some local temporary object so it cannot be returned by T1 const&, so we need to use plain T1.

Approach 3: Adding 3rd template argument for the return type
    template <typename T1, typename T2, typename RT>
    inline RT max (T1 const& a, T2 const& b) {
        return a < b ? b : a;
    }

    int main() {
        max<int,double,double>(4,4.2); // OK, but tedious
        max<double>(4,4.2); // OK: return type is double,T1 and T2 are deduced to int and double.
    }

B. Overloading the Functional Template:
    // maximum of two int type values
    inline int const& max (int const& a, int const& b) {
        return a < b ? b : a;
    }
    // maximum of two values of any type
    template <typename T>
    inline T const& max (T const& a, T const& b) {
        return a < b ? b : a;
    }
    // maximum of two char values
    inline char const& max (char const& a, char const& b) {
        return a < b ? b : a;
    }

    int main() { 
        ::max(7, 42);     // calls the nontemplate for two ints - Overload resolution.
                         // Prefers nontemplate version over one generated from the template.
        ::max('a', 'b');  // calls max<char> (by argument deduction) not non-template                            
                          // for two char bcos it comes too late definition
        ::max<>(7, 42);   // calls max<int> (by argument deduction)- <> indicates that only
                          // templates resolve a call rather than nontemplate version                          
        ::max('a', 42.7); // calls the nontemplate for two ints - Bcos auto type conversion
                          // is not supported for templates but OK for ordinary functions
    }

C. Nontype Function Template Parameters:
    template <typename T, int VAL> // int VAL is - Nontype Function Template Parameter
    T addValue (T const& x) {
        return x + VAL;
    }

These kinds of functions are useful if functions are used as parameters. For example,
    std::transform (source.begin(), source.end(),  // start and end of source
                    dest.begin(),                  // start of destination
                    addValue<int,5>);              // operation
The function -addValue - is called for each element in the source, while it is translated into the destination with +5.

Note: The above transform will give Compilation Error for the usage of templates function addValue<int,5>. Because function templates can be possibly overloaded but as per the C++ standard, overloadable functions cannot be used for template parameter deduction. So to solve this problem the template function should cast to exact type - (int(*)(int const&)) - to prove the uniqueness.
    std::transform (source.begin(), source.end(),          // start and end of source
                    dest.begin(),                          // start of destination
                    (int(*)(int const&)) addValue<int,5>); // operation

D. Using String Literals as an Arguments for the Function Templates:
Passing string literal by reference in function template will not always give the expected result. For string literals, there will be array-to-pointer conversion during argument deduction if and only if the parameter is not a reference.
    template <typename T> // note: reference parameters
    inline T const& max (T const& a, T const& b) { return a < b ? b : a; }

    int main() { 
        ::max("apple","peach");  // OK: same type
        ::max("apple","tomato"); // ERROR: different types
    }
Because "apple" and "peach" have type char const[6] whereas "tomato" has type char const[7]. On the first, both the type are the same so we got the expected result but in the second case the types are different but the template expects the same type. However, if you use non-reference parameters then internal array to pointer conversion will take place to solve this type of mismatch problem
    template <typename T> // Note: non reference parameters
    inline T max (T a, T b) { return a < b ? b : a; } 

    int main() {
        ::max("apple","peach");  // OK: same type
        ::max("apple","tomato"); // OK: Array-to-pointer conversion occurs.
    }

Understanding const char* Vs char[] : 
    template <typename T1, typename T2>
    void Ref_NonRef(T1 const& x, T2 y) {
        std::cout << "x in Ref(T1 const&): " << typeid(x).name(); // char [6]
        std::cout << "y in nonref(T2): "     << typeid(y).name(); // const char *
    }

    int main() {
        Ref_NonRef("hello","hello");
    }

     O/P:
            x in Ref(T1 const&) : char [6]
            y in nonref(T2)     : const char *

Member function template inside the class template:
    template <typename T>
    class List {                     // a namespace scope class template
    public:
        template <typename T2>       // a member function template
        List (List<T2> const&);      // (constructor)
    };
    
    template <typename T>
    template <typename T2>
    List<T>::List (List<T2> const& b) // an out-of-class member function template definition
    {                                
    }

Default Arguments with Function Templates:
Function templates can have default call arguments just like an ordinary function declarations
    template <typename T>
    void fill (Array<T>*, T const& = T());  // T() is zero for built-in types

Virtual Function Vs Member Function Templates:
Member function templates cannot be declared virtual. This constraint is imposed because the usual implementation of virtual function's call mechanism uses a fixed-size table with one entry per virtual function. However, the number of instantiations of a member function template is not fixed until the entire program has been translated. Hence, supporting virtual member function templates is not possible.

    template <typename T>
    class Dynamic {
    public:
        virtual ~Dynamic();   // OK: one destructor per instance of Dynamic<T>
        template <typename T2>
        virtual void copy (T2 const&); // ERROR: unknown number of instances of copy() 
// for a given instance of Dynamic<T>                      
    };
