For signed datatype, out of range is undefined.
For unsigned datatype, out of range datatype value wrap around.
    unsigned char a = 255; // a = 255
    unsigned char b = 256; // b = 0 (max = 255) - wraps around
    unsigned char c = 257; // c = 1 (257 - 256 = 1)
    unsigned char d = -1 ; // d = 255 - max - NOT wraps around
    unsigned char e = -2 ; // e = 254 - one number before max
Math:
The number 280 is too big to fit in our 1-byte range of 0 to 255. Greater tehan the largest number of the type is 256. Therefore, we divide 280 by 256, getting 1 remainder 24. The remainder of 24 is what is stored.

What’s the difference between a variable’s scope, lifetime, and linkage?
 - Scope determines where a variable is accessible. 
 - Lifetime determines where a variable is created and destroyed. 
 - Linkage determines whether the variable can be exported to another file or not.
    // External global variable [definitions]:
    int g_x;                       // defines non-initialized external global variable (zero initialized by default)
    extern const int g_x{ 1 };     // defines initialized const external global variable
    extern constexpr int g_x{ 2 }; // defines initialized constexpr external global variable
    
    // Forward [declarations]:
    extern int g_y;                // forward declaration for non-constant global variable
    extern const int g_y;          // forward declaration for const global variable
    extern constexpr int g_y;      // ERROR: not allowed: constexpr variables can't be forward declared
///////////////////////////////////////////////////////////////////////////////////////////////////
What kind of scope, duration, and linkage do global variables have?
 - Global variables have global scope (aka. file scope), which means they can be accessed from the point of declaration to the end of the file in which 
    they are declared.
 - Global variables have static duration, which means they are created when the program is started, and destroyed when it ends.
 - Global variables can have either internal or external linkage, via the static and extern keywords respectively.
///////////////////////////////////////////////////////////////////////////////////////////////////
summary: https://www.learncpp.com/cpp-tutorial/scope-duration-and-linkage-summary/
Scope:
    Variables with block scope / local scope can only be accessed within the block in which they are declared (including nested blocks)
    Variables and functions with global scope / file scope can be accessed anywhere in the file. 

Duration / Lifetime :
    Variables with automatic duration are created at the point of definition, and destroyed when the block they are part of is exited.
    Variables with static duration are created when the program begins and destroyed when the program ends.

Linkage:
- An identifier with 'no linkage' means the identifier only refers to itself. This includes:
Local variables
User-defined type definitions (such as enums and classes) declared inside a block
- An identifier with 'internal linkage' can be accessed anywhere within the file it is declared. This includes:
Static global variables (initialized or uninitialized)
Static functions
Const global variables
Functions declared inside an unnamed namespace
User-defined type definitions (such as enums and classes) declared inside an unnamed namespace
- An identifier with 'external linkage' can be accessed anywhere within the file it is declared, or other files (via a forward declaration). This includes:
Functions
Non-const global variables (initialized or uninitialized)
Extern const global variables
Inline const global variables
User-defined type definitions (such as enums and classes) declared inside a namespace or in the global scope

Inheritance: An object of one class can acquire the properties of an object of another class.

Object slicing: This happens when a derived class object is assigned to a base class object, additional attributes of a derived class object are sliced off to form the base class object.

In C++, a derived class object can be assigned to a base class object, but the other way is not possible.

    class Base { int x, y; };
    class Derived : public Base { int z, w; };

    int main() 
    {
        Derived d;
        Base b = d; // OK (base <= derived ) - Object Slicing,  z and w of d are sliced off

        // d = b;  // Error: (derived <= base) NOT allowed
    }



Virtual Function: A virtual function is a function, which forces the compile at the run time, to refer the function based on the content (address) of the base pointer, rather than the type of the base pointer.

    class Base {
    public:
        void display()      { cout<<"Base-display"; }
        virtual void show() { cout<<"Base-show"; } // Late Binding
    };

    class Derived : public Base {
    public:
        void display()   { cout<<"Derived-display"; }
        void show()      { cout<<"Derived-show"; }
        void derOwnFun() { cout<<"DerivedOwnFunction"; }
    };


    void main() {

        Base b, *bptr;
        Derived d;

        /////////////////  Using Pointers /////////////////////////////////////////////////////////////
        bptr = &d;         // points to derived but the type is Base
        // bptr = new Derived(); // same as above
        bptr->display();   // Base-display
        bptr->show();      // Derived-show - Resolved only by late binding using the Virtual keyword
        // bptr->derOwnFun(); // Error: Only inherited functions can be called using base pointer
        ((Derived*)bptr)->derOwnFun(); // OK: Explicit typecast for calling derived own function using base ptr 

        // Object slicing
        Base b = *bptr; // creates a new object 'b' by Object slicing of the content pointed by *bptr
        b.display();    // Base-display - 'b' lost all info about the derived class bcos of object slicing
        b.show();       // Base-show

        Base& b = *bptr; // NOT object slicing - It is reference.
        b.display();     // Base-display
        b.show();        // Derived-show 

        //////////////// Using Reference //////////////////////////////////////////////////////////////
        Base& bref = d;
        bref.display();    // Base-display
        bref.show();       // Derived-show
    }

Summary:

    void main() {
        Derived d;
        Base b = d;              // Base-show;    Base-display (Object Slicing)

        Base& b = d;             // Derived-show; Base-display (reference LateBinding)
        // Base b = &d; // Error 
        Base* b = &d;            // Derived-show; Base-display (pointer LateBinding)
        Base* b = new Derived(); // Derived-show; Base-display (pointer LateBinding)
    }


Abstract Class and Pure Virtual Function:
    class Base { public: virtual void display() = 0; };

    class D1 : public Base {
        public: void show() { cout << "Derived-show"; }
    };

    class D2 : public Base {
        public: void display() { cout << "Derived-display"; }  // Late Binding
    };

    void main()
    {
        // Base b; // Error: B has pure virtual function ie., Abstract Class
        // D1 d1;  // Error: D1 doesn't have the definition for pure virtual function so it will be change
                   // to Abstract Class because of inheriting the Abstract Class - Base
        D2 d2;     // OK - D2 has definition for the pure virtual function
        d2.display();
    }

Overloading Vs Overriding Vs Name Hiding:

    class Base {
    public:
        ~Base() {}
        Base() {}

        void f(int)    { cout << "Base::Int!" ; } // #1  
        void f(double) { cout << "Base::Double!" ; }    
    };

    class Derived : public Base {
    public:
        ~Derived() {}
        Derived() {}
        // brings the base class scope inside the derived class
        using Base::f; // #2  Explicit overloading resolution between 'Base' and 'Derived' for selecting 
// appropriate function.
        void f(double) { cout << "Derived::Double!" ; } // #3
    };

    int main(int, char **) {
        Derived d;
        d.f(21.21);
        d.f(21); 
        return 0;
    }


Output: 
Nothing Removed => Derived::Double! Base::Int!
If #x is removed
#1              => Derived::Double! Derived::Double! 
#2              => Derived::Double! Derived::Double! (In C++, there is no overloading across inheritance by default)
#3              => Base::Double!    Base::Int!
#2 and #3       => Base::Double!    Base::Int!

Note: By specifying the 'using' declaration, you bringing all of the overloads of 'f' from 'Base' into the scope of 'Derived'. Then overloading will work properly.


Example 2: Important !!!

    class Base {
    public:
        void bC(char c)    { std::cout << "Base::bC(char c)" << c << std::endl; }
        void bdFC(float f) { std::cout << "Base::bdFC(float x)" << f << std::endl; }

        virtual void vbdC(char c)   { std::cout << "Base::vbdC(char c)" << c << std::endl; }
        virtual void vbdFC(float c) { std::cout << "Base::vbdFC(float c)" << c << std::endl; }
    };

    class Derived : public Base {
    public:
        void bdFC(char c)  { std::cout << "Derived::bdFC(char c)" << c << std::endl; }

        void vbdC(char c)  { std::cout << "Derived::vbdC(char c)" << c << std::endl; }
        void vbdFC(char c) { std::cout << "Derived::vbdFC(char c)" << c << std::endl; }
    };

    int main() {
        Derived* d = new Derived();
        d->bC(65.37f);    // Base::bC(char c) A - type conversion [float->char]. A - ASCII of 65
        d->bdFC(65.37f);  // Derived::bdFC(char c) A; Base class bFdC(float f) hidden by derived class bFdC(char c)
        d->vbdC('B');     // Derived::vbdC(char c) B
        d->vbdFC(65.37f); // Derived::vbdFC(char c) A (NOT refers Base bcos 'd' is NOT polymorphism object )

  // Non-Virtual Function caller preferred by 1. static/declared type of function implementation
  // Virtual function caller preferred by 1. dynamic/actual type of function implementation and 2. argument type

        Base* b = d;
        b->bdFC(65.37f);  // Base::bdFC(float x) 65.37 - NO virtual function involved
        b->bdFC('B');     // Base::bdFC(float x) 66 - type conversion [char->float] - NO virtual function involved.
        b->vbdC(65.37f);  // Derived::vbdC(char c) A - Important!!
        b->vbdFC(65.37f); // Base::vbdFC(float c)65.37 - eventhough it has virtual function but its param are different
    }



Note: 
1. Function with default parameters mostly bring the ambiguous issue. Ex: void foo(int x = 2) {} will ambigous to both void foo() {} and void foo(int x) {}
1. Function can NOT be overloaded by return type. Can overload by 'const'.
2. Function name hiding across the inheritance can happen by 1. return type, 2. parameters and 3. const 

Name Hiding: 
Non-virtual function in the derived class hides the same name non-virtual functions in the base class regardless of 1. argument list, 2. return type and 3. const.
        Solution: Bring the problematic function from base to derived class scope by defining " using Base::foo; "  inside the derived class
Virtual function has no such name hiding issue.


Empty Class:

If you create an object for an Empty class then the compiler will provide the default version of the following functions if they are not explicitly declared.

    class X {
        // 6 are member functions that the compiler can generate
        X();                     // Default constructor
        ~X();                    // Destructor
        X(const X&);             // Copy constructor
        X(X&&);                  // Move Constructor
        X& operator=(const X&);  // Copy assignment operator
        X& operator=(X&&);       // Move assignment operator
    };
