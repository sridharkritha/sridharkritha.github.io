LTF = Local To Function

caller            callee
----------------------------------------------
1.  x     x - Pass by Value (LTF)
2. &x    *p - Pass by address
3.  x    &x - Pass by reference (C++)
4.  p    *p - Pointer by Value (LTF)
5. &p   **p - Pointer to Pointer
6.  p   *&p - Reference to Pointer (C++)
7.  a   a[] / *a - both are the same
8.  v   v[][7] / (*v)[7]/ **v - both are the same. Note: (*v)[7] - Called as "Pointer to an Array"
9.  q   **q - Array of Pointer

Note: In C/C++ array can be passed either as a pointer(*a) or reference(a[]) and not by value(a). So if the called function makes any changes in the array it will reflect to the caller array content.

A. Pass by Value:

    }

    int main(void) {
        int k=10;
        foo(k);   /*  k still equals 10  */
    }
 
B. Pass by Address (reference) - C Style:
    void foo(int *j) {
        *j = 0;
    }

    int main(void) {
        int k=10;

C. Pass by reference - C++ Style:

    }

Pass by pointer: When you pass a pointer to a function, only a copy of the pointer is passed to the function. When you modify the pointer inside the function you are only modifying a copy of the pointer and the original pointer remains unmodified. We can say "pass by pointer" is passing a "pointer by value".
    int g_One = 1;         // global variable
    void func(int *pInt, int *qInt); // function prototype

    int main() {
        int two  = 2;
        int four = 4;

        int *pTwo = &two;
        func(pTwo, &four);

        std::cout << two << *pTwo << std::endl; //  2 2 (NO change)
        std::cout << four << std::endl;         //  1   (Updated)
    }

    void func(int *pInt, int *qInt) {

        *qInt = g_One;  // PASS by address
    }

Pointer to Pointer:
Any changes done on the content pointed by the pointer inside the function will reflect on the original content pointed by another alias pointer.
    int g_One = 1;
    void func(int** ppInt); // function prototype

    int main() {
        int nvar  = 2;
        int *pvar = &nvar;
        func(&pvar);
        cout << nvar << *pvar; // 2 1
    }

    void func(int** ppInt) {
        *ppInt = &g_One;  // Replace the source content LOCATION  
        // or
        *ppInt = new int; //  Replace the source content LOCATION by allocating a new memeory location
        **ppInt = 1;      // and put the new content 1 on the new location 
    }
Let me summarise what all those dereferencing are,
    - ppInt is the ptr-to-ptr. We will never modify this because if we do, we'll lose our handle on the
       address of the pointer it is pointing to.
    - *ppInt is the pointed pointer. If we modify this, we are modifying the contents of the pointed
       pointer, which is an address.
    - **ppInt is dereferenced twice to value which is what pvar points to.

Reference to Pointer: Please note we can use pass by pointer, pointer to pointer in both C and C++ but we can use reference to pointer only in C++.

Returning Arrays from Function:
    // Int to String Conversion - itoa : PROBLEM
    char *itoa(int n) {
        char retbuf[25];
        sprintf(retbuf, "%d", n);  // Instead of using do/while
        return retbuf;    // Logical Error - stack memory array should not be returned
    }

    // Solution 1: Static Storage 
    char *itoa(int n) {
        static char retbuf[25];
        sprintf(retbuf, "%d", n);  // Instead of using do/while
        return retbuf;    // OK - static array can be returned
    }

Drawback:  Each time you call the function, it re-uses the same array and returns the same pointer. Therefore, when you call the function a second time, whatever information it ``returned'' to you last time will be overwritten.
    printf("i = %s, j = %s\n", itoa(20), itoa(40));  // 40 40 (or) 20 20

The moral is that although the static return array technique will work, the caller has to be a little bit careful, and must never 
expect the return pointer from one call to the function to be usable after a later call to the function.
