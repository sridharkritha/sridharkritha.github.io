DataTypes



Numeral Automatic Type Conversion: Arithmetic operators require their operands to be of the same type. If operands of mixed types are used, the compiler will convert one operand of a smaller data type to a larger data type.
 
Hierarchy of data types:

Floating point promotion:
    long double
    double
    float (Lowest)

Integeral Promotion:
    unsigned long long
    long long
    unsigned long int
    long int
    unsigned int   <- [char, unsigned char, unsigned short]
    int (lowest)   <- [char, signed char, signed short, bool]

Arithmetic operators do not accept types smaller than int as arguments, and integral promotions are automatically applied. This conversion always preserves the value.

signed char or signed short can be converted to int.
unsigned char or unsigned short can be converted to unsigned int.
char can be converted to int or unsigned int depending upon the expression
bool can be converted to int with the value false becoming ​0​ and true becoming 1.

Note:
A raw number without floating point considered as signed interger. Ex: 50 is a signed integer by default but 50U is a Unsigned integer.
A raw number with floating point considered as double. Ex: 3.14 is a double by default but 3.14F is a float.

Win32 API based data types:
There are 3 encoding schemes and 3 character types. The first scheme is the single-byte character set, or SBCS. In this encoding scheme, all characters are exactly one byte long. ASCII is an example of an SBCS. A single zero byte marks the end of a SBCS string. The second scheme is the multi-byte character set, or MBCS. An MBCS encoding contains some characters that are one byte long, and others that are more than one byte long. The third scheme is Unicode. Unicode is an encoding standard in which all characters are two bytes long. Unicode characters are sometimes called wide characters because they are wider (use more storage) than single-byte characters. Note that Unicode is not considered an MBCS - the distinguishing feature of an MBCS encoding is that characters are of different lengths. A Unicode string is terminated by two zero bytes (the encoding of the value 0 in a wide character).

    1. single-byte character set, or SBCS   -  all characters are exactly 1 byte long.
    2. multi-byte character set, or MBCS   -  some are 1 byte and other are more than 1 byte long.
    3. Unicode /  Wide character set         -  all characters are exactly 2 bytes long.

    wchar_t  wch = L'1';      // 2 bytes, 0x0031
    wchar_t* wsz = L"Hello";  // 12 bytes, 6 wide characters

TCHAR to the rescue!

TCHAR is a character type that lets you use the same codebase for both MBCS and Unicode builds, without putting messy #defines all over your code. A definition of the TCHAR looks like this:

    #ifdef UNICODE
        typedef wchar_t TCHAR;
    #else
        typedef char TCHAR;
    #endif

So a TCHAR is a char in MBCS builds, and a wchar_t in Unicode builds. There is also a macro _T() to deal with the L prefix needed for Unicode string literals:

    #ifdef UNICODE
        #define _T(x) L##x
    #else
        #define _T(x) x
    #endif

The ## is a preprocessor operator that pastes the two arguments together. Whenever you have a string literal in your code, use the _T macro on it, and it will have the L prefix added on when you do a Unicode build.

    TCHAR szNewText[] = _T("we love Bob!");

8. Use of  (L,U,F) or (l,u,f) suffixes on numeric literals (3.14)?

F or f:
    float pi = 3.14f
By default the float numeric literals(3.14) considered as a double type, unless use 'f' or 'F' suffix to indicate it's type is float.

U or u :
    unsigned int i = 786U;
Otherwise, 786 will be considered as signed int  by default .

L or l:
    long int i = 786L;
Explicitly forcing the compiler that to use long arithmetic while using the long int i.

    #define SECONDS_PER_YEAR   (60 * 60 * 24 * 365)UL
This expression won't be suitable for an integer argument on a 16-bit machine-hence the need for the L, telling the compiler to treat the variable as a Long.

9. what is meant by signed char ?
char is an integer type, same as int, short and other integer types. 'char' just happens to be the smallest integer of size always 1 byte. Just like any other integer type, it can be signed or unsigned (default).

Only the main difference between char and other integer types is that plain char is NOT signed char by default, while with other integer types the signed  by default.

So char can be represented by 3 ways 1. char 2. signed char 3. unsigned char.

By coding standard, the plain char is mostly used for character representation and signed/unsigned char used for 1 byte integer.
