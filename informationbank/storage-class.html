Storage Class



A. Storage Class: 'Storage' refers to the scope of a variable and where the memory allocated by the compiler to store that variable.

Types of Storage Classes: Storage classes are categorised into 4 types as,
    1. Automatic Storage Class
    2. Register Storage Class
    3. Static Storage Class
    4. External Storage Class

Automatic Storage Class :
    Keyword : auto
    Storage Location : stack segment
    Initial Value : Garbage Value
    Life : Control remains in a block where it is defined.
    Scope : Local to the block in which the variable is declared.

Example :  auto int a;

Register Storage Class :
    Keyword : register
    Storage Location : CPU Register
    Initial Value : Garbage
    Life : Local to the block in which variable is declared.
    Scope : Local to the block.

Example : register int a;

Static Storage Class :
    Keyword : static
    Storage Location : Uninitialized static variables are stored in BSS segment, Initialized static - in Data Segment.
    Initial Value : Zero and can be initialized only once.
    Life : depends on function calls and the whole application or program.
    Scope : Local to the block.

Example : static int a;

External Storage Class :
    Keyword : extern
    Storage Location : Main memory
    Initial Value : Zero
    Life : Until the program ends.
    Scope : Global to the program.

The variables of this class can be referred to as 'global or external variables.' They are declared outside the functions and can be invoked anywhere in a program.

Example :  file1.cpp => int a;  - stored in global heap memory
                 file2.cpp => extern int a;
                 file3.cpp => extern int a;

B. The Preprocessor: Preprocessor is a program that processes the code before it passes through the compiler. Preprocessor operates on preprocessor command and directives.

The preprocessor directives can be divided into 3 categories
    1. File inclusion directives
    2. Macro substitution directives
    3. Compiler control directives

1. File inclusion directives: This can be used to include any file in place of the directive in your program.
#include"myheader.h" - Preprocessor searches in the same directory (local) as the source file. if not found then looks for standard directory 
#include<stdio.h>     - looks for standard directory

2. Macro substitution directives: There are different forms of macro substitution. The most common form is
    1. Simple macro substitution
    2. Argument macro substitution
    3. Nested macro substitution

a. Simple macro substitution: Simple string replacement is commonly used to define constants
Example: #define PI 3.1415926

b. Argument macro substitution: The preprocessor permits us to define the complex and more useful form of replacements
# define CUBE (x) (x*x*x)

c. Nested macro substitution
#define AREA(r) (PI*r*r)

#define PI 3.1415926

3. Compiler control directives: Enable the programmer to control the compilation of program code.

Examples:
#if !defined BUFFER_H__
#define BUFFER_H__ 
   Header code related to buffer
#endif


#if   !defined NULL
#define NULL 0
#endif

#if  0
    code prevented  from  compiling...
#endif

Other Advance Macros:

Pasting Tokens Macros:
Each argument passed to a macro is a token. To paste tokens in a macro, use ## between the two things to paste together.

#define BUILD_FIELD(field) my_struct.inner_struct.##field

BUILD_FIELD(data)  => my_struct.inner_struct.data

Macros contain if statement:

Macro Definition: - Wrong

 #define MYMACRO(a,b) \               
     if (xyzzy) asdf()      
     
Macro Caller:
   
if (exp)
     MYMACRO(foo,bar);
 else
     function(f00);      
     
Macro Expansion:

if (exp)
     if (xyzzy) asdf();
     else
     function(f00);  // Logical error - else belongs to if(exp) not to if (xyzzy)
     
Solution:
Method 1. Adding dummy else statement (recommended solution):

Macro Definition:
#define MYMACRO(a,b) \              
     if (xyzzy) asdf(); \
     else (void)0

Note: Last line should not end with semi colon bcos semicolon will be attached by macro caller.

Macro Expansion:
if (whatever)
     if (xyzzy)
         asdf();
     else
         (void)0;
 else    ^^^^^^^^—that's a do-nothing statement
     function(f00);

Method 2. Adding do - while(false) statement:

Macro Definition:
#define MYMACRO(a,b) \               
     do { \
       if (xyzzy) asdf(); \
     } while (false)

Macro contains multi-line statements:

Macro Definition:-Wrong

#define MYMACRO(a,b) \              
     statement1; \
     statement2; \
     ... \
     statementN;

Macro Caller:

if (whatever)
     MYMACRO(foo, bar);
 else
     function(f00);

Macro Expansion:
if (whatever)
     statement1;
     statement2;   // statement 2 comes out of if
     ...
     statementN;
  ; else
 ^^^^^^^^—compile-time error!
     function(f00);
     
Solution:
1. Add the braces outside the multiline statements
2. avoid semicolon at the last line and
3. include single run loop constraint
Method 1:
Macro Definition:

#define MYMACRO(a, b) \
     if (true) { \
         statement1; \
         statement2; \
         ... \
         statementN; \
     } else
         (void)0     <-intentionally not including ;

Macro Expansion:

if (whatever)
     if (true) {
         statement1;
         statement2;
         ...
         statementN;
     } else
         (void)0;
 else    ^^^^^^^^—a do-nothing statement
     function(f00);

         
Method 2:
Macro Definition:

 #define MYMACRO(a, b) \                (Okay)
     do { \
         statement1; \
         statement2; \
         ... \
         statementN; \
     } while (false)      <-intentionally not including ;

#define SWAP(a, b)
