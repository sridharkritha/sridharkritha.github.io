Shallow Vs. Deep Copying


this Pointer:

this - is a pointer variable that always contains the address of the invoking object.
(*this) - Refers to the object pointed by 'this' pointer. That is '*this' can be considered as the object itself.
"this-> " - Another form of representing "*this"
* Data type of this pointer for a member function of a class type X, is X* const this;
* Data type of this pointer for a const member function of a class type X, is const X* const this;
* A static member function does not have a this pointer.
    class test {
    public:
        int var;
        test(int x)
        {
            // this - Implicit object
            (*this).var = x;  //  obj = (*this) = this->
            cout << this->var;
        }
        // Copy Constructor
        test(test const& c)  // Note: test(test c) give a compilation error: & must   
        {           
            cout<<c.var;  // c - Explicit object
        }
    };

    int main()
    {
        test a(5);
        test b(a);
    }
Advantage: A more practical use of 'this' pointer is in returning values from the member function. when an object is local to the function, the object will be destroyed when the function terminates. It necessitates the need for a more permanent object while returning it by reference.
    complex& complex::add(complex c2)
    {
        real = real + c2.real; // adds the c2 object real with invoking object real
        img = img + c2.img;
        return complex(real, img); // Explicitly creating a nameless object using the constructor 
    }
This can be rewritten by using '*this' :
    complex& complex::add(complex c2)
    {
        real = real + c2.real; 
        img = img + c2.img;
        return *this; // *this - obj which has updated value (real,img - data memebers)
    }


Overloading the Assignment Operator: The assignment operator is used to copy the values from one object to another already existing object. The keyword here is “already exist". Consider the following example: 

    Cents cMark(5); // calls Cents constructor
    Cents cNancy;   // calls Cents default constructor
    cNancy = cMark; // calls Cents assignment operator

 In this case, cNancy has already been created, So the assignment operator must be overloaded as a member function.

 

Copy Constructor: What happens if the object being copied does not already exist? To understand consider the following example: 

    Cents cMark(5);       // calls Cents constructor
    Cents cNancy = cMark; // calls Cents copy constructor!

Note in this case, cNancy is not already created object. Because the second statement uses an equal’s symbol in it, you might expect that it calls the assignment operator. However, it doesn’t! It actually calls a special type of constructor called a copy constructor. A copy constructor is a special constructor that initializes a new object from an existing object.


Copy Constructor Vs Assignment Operator:

    * If a new object has to be created before the copying can occur, the copy constructor is used.

    * If a new object does not have to be created before the copying can occur, the assignment operator is used.

      

 There are three general cases where the copy constructor is called instead of the assignment operator: 

   1. When instantiating one object with values from another object (as in the example above).

   2. When passing an object by value to the constructor.

   3. When an object is returned from a function by value.


  • When an object is created from another object during initialization [ Class a = b ]
  • When an object is created from another object as a parameter to a constructor [ Class a(b) ]
  • When an object is passed by value as an argument to a function [ function(Class a) ]
  • When an object is return from a function [ Class a; fun() { return a; } ]
  • When an exception is thrown using this object type [ Class a; fun() { throw a; } ] 

Because the copy constructor and assignment operator essentially do the same job, the code needed to implement them is almost identical. 

Overloaded Assignment operator and Copy Constructor

    class Cents
    {
    private:
        int m_nCents;

    public:
        Cents(int nCents=0) { m_nCents = nCents; } 
        Cents(const Cents& cSource) { m_nCents = cSource.m_nCents; }  // Copy constructor
        Cents& operator= (const Cents& cSource); 
    };

    // Overloaded = operator
    Cents& Cents::operator= (const Cents& cSource)
    {
        /* check for self-assignment by comparing the address of the
        implicit object and the parameter */
        if (this == &cSource) return *this; 

        m_nCents = cSource.m_nCents; // do the copy

        return *this; // return the existing object
    } 

 

A couple of things to note here:  

 

1. In Copy constructor parameter MUST be passed by reference, and not by value. Can you figure out why? Because a copy constructor is called when a parameter is passed by value. If we pass our cSource parameter by value, it would need to call the copy constructor to do so. But calling the copy constructor again would mean the parameter is passed by value again, requiring another call to the copy constructor. This would result in an infinite recursion. 

 

2. In Assignment Operator overloading, we’re returning *this so we can chain multiple assignments together. For example 

    cMark = cNancy = cFred = cJoe;  // assign cJoe to everyone

 

3. In C++, it is possible to do a self-assignment like

    cMark = cMark;                  // valid assignment

   So if the class uses dynamic memory, it can be dangerous. It is a good idea to do a check for self-assignment at the top of an overloaded assignment operator. But for the case of copy-constructor self-assignment check is not needed because the copy constructor is only called when new objects are being constructed, and there is no way to assign a newly created object to itself in a way that calls to copy constructor.


4. How do I specify a copy constructor for a class whose code is not accessible?  Use the keyword operator.

    // suppose class A's code is not accessible
    A::A(const int i) { /* ... */ }

    // This does not prevent you to make a B to A conversion
    struct B
    {
        int b;
        B(const int i): b(i) {}
        operator A() { return A(b); }   // implicit type conversion using the keyword operator
    };


One more Example:

    class Base {
    public:
        Base(int initialValue = 0): x(initialValue) {}
        Base(const Base& rhs): x(rhs.x) {} // Base copy constructor
    private:
        int x;
    };

    class Derived: public Base {
    public:
        Derived(int initialValue): Base(initialValue), y(initialValue) {}
        Derived& operator=(const Derived& rhs);             // Derived Assignment operator
        Derived(const Derived& rhs): Base(rhs), y(rhs.y) {} // Derived copy construtor
    private:
        int y;
    };

    // Assignment operator
    Derived& Derived::operator=(const Derived& rhs)
    {
        if (this == &rhs) return *this;

        // Base part of *this (bcos of &, *this just a ref to base obj not really changed into base object)
        // Method - I
        static_cast<Base&>(*this) = rhs; // call Base operator= 

        // Methoid - II - but some compiler throws error    
        Base::operator=(rhs);           // call this->Base::operator=  

        // Derived part of *this
        y = rhs.y;
        return *this;
    }

    void main()
    {
        Derived d1(0);   // d1.x = 0, d1.y = 0
        Derived d2(1);   // d2.x = 1, d2.y = 1
        d1 = d2;         // d1.x = 1, d1.y = 1
        Derived d3 = d2; // d1.x = 1, d1.y = 1
    }


Object Creation :

    Foo obj(args);        // Creates one Foo object and calls one argument constructor once.

    Foo obj2;             // Creates one Foo object and calls no arg / default constructor once.
    obj2 = Foo(args);      // a temporary object creation on the RHS and then call operator= copy-assignment

    Foo obj3 = Foo(args); // Two objects creation, one is temporary and other is obj3 and then calls copy constructor.
        // or
    Foo obj3(Foo(args));

    class B {
    public:
        B(int i)      { cout<<"B Single Arg Constructor: "<<i<<endl; }  
        B()           { cout<<"B Constructor\n"; }
        B(B const& o) { cout<<"B Copy Constructor: "<<endl; }
        B& operator=(B const& o) { 
                cout<<"B Overloaded assignment operator: "<<endl;
                return *this;
        }
        ~B()    { cout<<"B Destructor\n"; }    
    };

    int main(){
        B b;      // B Constructor   B Destructor
        B b(5);   // B Single Arg Constructor: 5      B Destructor

        B b();    // No output: b() consider as a function returning type B and taking no arg        
        B b(B()); // No output: b() consider as a function returning type B and taking an argument of type pointer
                  // to function returning B taking no arguments.

        // Most vexing parse - use extra ()
        B b((B())); // B Constructor   B Destructor - Now the rvalue expression B() creates a temp obj[call to the 
                    // default constructor] and b is copy initialized to that temp obj[no call to the copy-ctor 
                    // because of some compiler optimization]
        // or
        B b = B();  // B Constructor   B Destructor
    }

Instance Initialization Syntax:
    class Foo {
    public:
        explicit Foo(int);
        Foo& operator=(int);
    };
    Foo f(10);      // "direct initialization" - works
    Foo f = 10;     // "copy initialization" - doesn't work
    Foo f(Foo(10)); // "copy initialization" - doesn't work. Also equivalent to: Foo f = 10;
The difference is that the conversion constructor cannot be used for implicit conversions when you put explicit before it.
But

    Foo f;
    f = 10; // works
Above will use the default constructor of Foo (the one taking no arguments) not assignment operator.
