OOPs Concepts:

Class: Class is a basic construct for creating user-defined data types. It also explains the structure of an Object.

Object: Instance of Class

Encapsulation: Wrapping up data and function into a single unit (called class) is known as encapsulation.

Data Hiding : Insulation of the data from direct access by the program is called data hiding. Only those functions which are wrapped in the class can access it. Ex : public, protected, private.


Inheritance :

It is a method by which an object of one class can acquire the properties of an object of another class.

The concept of inheritance provides the idea of reusability, this means that we can add additional features to an existing class without modifying it.


Polymorphism:

Polymorphism is a concept of using “operators “ and “functions” in different ways, depending on what they operate on. Ex: function overloading, operator overloading, and virtual function. 

Note: Polymorphism works on references (the result of dereferencing a pointer) just as it does on pointers.

Types:

1. Compile time polymorphism [Linking takes place at compile time - Early binding / Static binding]

1. Function overloading

2. Operator Overloading

2. Run time polymorphism [Linking takes place after compile time - Late binding / Dynamic binding]

1. Virtual Function


Late Binding:

To cause late binding to occur for a particular function, C++ requires that you have to use the virtual keyword when declaring the function in the base class. Late binding occurs only with virtual functions, and only when you’re using an address of the base class where those virtual functions exist, although they may also be defined in an earlier base class.

Virtual Function:

 A virtual function is a function, which forces the compile at the run time, to refer the function based on the content (address/actual type/dynamic type) of the base pointer, rather than the declared type(static type) of the base pointer.

 A virtual function in a parent allows children to have a different implementation for it. A pure virtual method in a parent forces children to should have an implementation/definition for virtual methods.

Note: You are only required to declare a function virtual in the base class. All derived-class functions that redefine the virtual function of the base-class are usually called overriding mechanism. You can also use the virtual keyword in the derived-class declarations (it does no harm to do so), but it is redundant and can be confusing.

Ex:


    class Base {
    public:
        void display()      { cout << "Base-display"; }
        virtual void show() { cout << "Base-show"; } // Late Binding
    };

    class Derived : public Base {
    public:
        void display()   { cout << "Derived-display"; }
        void show()      { cout << "Derived-show"; }
        void derOwnFun() { cout << "DerivedOwnFunction"; }
    };

    void main() {

        Base b, *bptr;
        Derived d;

        /////////////////  Using Pointers /////////////////////////////////////////////////////////////
        bptr = &b;       // pointer points to base
        bptr->display(); // Base-display
        bptr->show();    // Base-show

        bptr = &d;         // points to derived but the type is Base
        // bptr = new Derived(); // same as above
        bptr->display();   // Base-display
        bptr->show();      // Derived-show - Resolved only by late binding using the Virtual keyword
        // bptr->derOwnFun(); // Error: Only inherited functions can be called using base pointer
        ((Derived*)bptr)->derOwnFun(); // OK: Explicit typecast for calling derived own function using base ptr 

        // Object slicing
        Base b = *bptr; // creates a new object 'b' by Object slicing of the content pointed by *bptr
        b.display();    // Base-display - 'b' lost all info about the derived class bcos of object slicing
        b.show();       // Base-show

        Base& b = *bptr; // NOT object slicing - It is reference.
        b.display();     // Base-display
        b.show();        // Derived-show 

        //////////////// Using Reference //////////////////////////////////////////////////////////////
        Base& bref = d;
        bref.display();    // Base-display
        bref.show();       // Derived-show
    }

Summary:

    void main() {
        Derived d;
        Base b = d;              // Base-show;    Base-display (Object Slicing)

        Base& b = d;             // Derived-show; Base-display (reference LateBinding)
        // Base b = &d; // Error 
        Base* b = &d;            // Derived-show; Base-display (pointer LateBinding)
        Base* b = new Derived(); // Derived-show; Base-display (pointer LateBinding)
    }


Object slicing: This happens when a derived class object is assigned to a base class object, additional attributes of a derived class object are sliced off to form the base class object.

In C++, a derived class object can be assigned to a base class object, but the other way is not possible.

    class Base { int x, y; };
    class Derived : public Base { int z, w; };

    int main() 
    {
        Derived d;
        Base b = d; // OK (base <= derived ) - Object Slicing,  z and w of d are sliced off

        // d = b;  // Error: (derived <= base) NOT allowed
    }

Ex 2:

    void foo(Base b)  // object slicing happens by "Base b = d;"
    { 
        // b knows only about Base members and lost all info about 'd' specific members 
    }

    Derived d;
    foo(d); // pass 'd' to 'b'

Abstract Class:

    * Abstract class is one that is not used to create objects.

    * Such a class exist only as a parent for the derived class. 

Pure Virtual / Abstract Function and Abstract Class:

* Pure virtual function is a “do-nothing “ function, which has no definition.

* This function is only to serve as a place holder ie., meant to be redefined by the derived classes. 

* Class which has at least one pure virtual function then this class cannot be instantiated ie., change to Abstract Class. Note: Abstract class shall have protected constructors rather than public because we are going to create obj only for the class which is inherited from abstract class rather than directly to the abstract class.

* Class which is derived from Abstract Class must have a definition for the pure virtual function otherwise we cannot instantiate the derived class that is derived class also will change to another Abstract Class.

* Pure virtual functions are always “equated to zero” (=0), for informing the compiler don't look for the definition part of the function.

  Ex: virtual void display()=0;

class Base {
public:
    string author;
    Base(string name) : author(name) { }
    virtual void display() = 0;
};

class D1 : public Base {
public:    
    virtual void show() { cout << "Derived-show"; }
};

class D2 : public Base {
public:
    D2(string name) : Base(name) { }
    void display() { cout << "Derived-display" << author; } // Late Binding
};

void main()
{
    Base B; // Error: B has pure virtual function ie., Abstract Class
    D1 d;   // Error: D1 doesn't have the definition for pure virtual function so it will be change
            // to Abstract Class because of inheriting the Abstract Class - Base

    D2 e("Sridhar"),f;  // Correct !! - D2 has definition for the pure virtual function
    f->display();
}


Interface Class:

An interface class is a class that has no members variables and all of the functions are pure virtual functions!. Interfaces are useful when you want to define the functionality that derived classes must implement, but leave the details of how the derived class implements that functionality entirely up to the derived class. Interface classes are often named beginning with an I. Here’s a sample interface class:

    Class IBase {
        virtual void display() = 0;
        virtual void show() = 0;
    };


Virtual Table:

The virtual table is a lookup table of functions used to resolve the function calls in a dynamic/late binding. 

    class Base {
    public:
        //Invisible ptr created by compiler which points to  ==>            " Base VTable "
        FunctionPointer *__vptr;     // points to Base VTable           -----------------------------
        virtual void display() { };  //                                        Base::display()
        virtual void show() { };     //                                        Base::show()
    };
        
    class D1: public Base {
    public:                          //                                        " D1 VTable "
        FunctionPointer *__vptr;     // points to D1 VTable               -----------------------------
        virtual void display() { };  //                                         D1::display()
    };                               //                                         Base::show()
        
    class D2: public Base {          //                                         " D2 VTable "
    public:                          //                                     -----------------------------
        FunctionPointer *__vptr;     //                                          Base::display()
        virtual void show() { };     //                                          D2::show()
    };

* If a class uses virtual functions (or is derived from a class that uses virtual functions) then it has an own virtual table. 

* A virtual table contains the collection of function pointers(D1::display()) that points to the most-derived functions with respect to corresponding virtual functions defined in the base class(Base::display()). 

* Additionally the compiler also adds a hidden pointer(*__vptr) to the base class which has a virtual function and the derived which overrides the base class virtual function. 

*__vptr will automatically point to the virtual table for that class when a class instance is created. 


Overriding:

The redefinition of a virtual function in a derived class is usually called overriding.

Overloading Vs Overriding:

Function Overloading is a feature that happens within the same class and function whereas Overriding is a feature that happens within the inheritance chain by using the virtual keyword for that function in the base class. Overriding is resolved during the run time and Overloading at compile time. Overriding means the function signature is the same and Overloading needs a unique function signature.

Note: Functional overloading is not supported in C but supported in C++. Normally the function overloading by return type is not possible.

    class Base {
    public:
        ~Base() {}
        Base() {}

        void f(int)    { cout << "Base::Int!" ; } // #1  
        void f(double) { cout << "Base::Double!" ; }    
    };

    class Derived : public Base {
    public:
        ~Derived() {}
        Derived() {}
        // brings the base class scope inside the derived class
        using Base::f; // #2  Explicit overloading resolution between 'Base' and 'Derived' for selecting 
// appropriate function.
        void f(double) { cout << "Derived::Double!" ; } // #3
    };

    int main(int, char **) {
        Derived d;
        d.f(21.21);
        d.f(21); 
        return 0;
    }


Output: 
Nothing Removed => Derived::Double! Base::Int!
If #x is removed
#1              => Derived::Double! Derived::Double! 
#2              => Derived::Double! Derived::Double! (In C++, there is no overloading across inheritance by default)
#3              => Base::Double!    Base::Int!
#2 and #3       => Base::Double!    Base::Int!

Note: By specifying the 'using' declaration, you bringing all of the overloads of 'f' from 'Base' into the scope of 'Derived'. Then overloading will work properly.


Example 2: Important !!!

    class Base {
    public:
        void bC(char c)    { std::cout << "Base::bC(char c)" << c << std::endl; }
        void bdFC(float f) { std::cout << "Base::bdFC(float x)" << f << std::endl; }

        virtual void vbdC(char c)   { std::cout << "Base::vbdC(char c)" << c << std::endl; }
        virtual void vbdFC(float c) { std::cout << "Base::vbdFC(float c)" << c << std::endl; }
    };

    class Derived : public Base {
    public:
        void bdFC(char c)  { std::cout << "Derived::bdFC(char c)" << c << std::endl; }

        void vbdC(char c)  { std::cout << "Derived::vbdC(char c)" << c << std::endl; }
        void vbdFC(char c) { std::cout << "Derived::vbdFC(char c)" << c << std::endl; }
    };

    int main() {
        Derived* d = new Derived();
        d->bC(65.37f);    // Base::bC(char c) A - type conversion [float->char]. A - ASCII of 65
        d->bdFC(65.37f);  // Derived::bdFC(char c) A; Base class bFdC(float f) hidden by derived class bFdC(char c)
        d->vbdC('B');     // Derived::vbdC(char c) B
        d->vbdFC(65.37f); // Derived::vbdFC(char c) A (NOT refers Base bcos 'd' is NOT polymorphism object )

  // Non-Virtual Function caller preferred by 1. static/declared type of function implementation
  // Virtual function caller preferred by 1. dynamic/actual type of function implementation and 2. argument type

        Base* b = d;
        b->bdFC(65.37f);  // Base::bdFC(float x) 65.37 - NO virtual function involved
        b->bdFC('B');     // Base::bdFC(float x) 66 - type conversion [char->float] - NO virtual function involved.
        b->vbdC(65.37f);  // Derived::vbdC(char c) A - Important!!
        b->vbdFC(65.37f); // Base::vbdFC(float c)65.37 - eventhough it has virtual function but its param are different
    }

Name Hiding and Overloading:

If you redefine an overloaded function name from the base class, all the other versions of the base class overloaded function are automatically hidden in the new derived class.

Overloaded name hiding during inheritance even can take place if the member function argument list or return type or const in the derived class is different from the base member function

    class Base {
    public:
        int f() const {
            cout << "int Base::f()\n";
            return 1;
        }
        int f(string) const { 
            cout << "int Base::f(string)\n";
            return 1; 
        }
        void g() const { cout << "void Base::g() const\n"; }
    };

    class Derived1 : public Base {
    public:
        // removed const:
        void g() { cout << "Removed const => Derived1::g() \n"; }
    };

//////////////////////////////////////// main ///////////////////////////////////////////////////
        string s("hello");

        Derived1 d1;       // NOTHING
        d1.g();            // Removed const => Derived1::g()
        int x = d1.f();    // int Base::f()
        d1.f(s);           // int Base::f(string)
/////////////////////////////////////////////////////////////////////////////////////////////////

    class Derived2 : public Base {
    public:
        // Redefinition:
        int f() const {
            cout << "Redefinition => Derived2::f()\n";
            return 2;
        }
    };
////////////////////////////////////////// main //////////////////////////////////////////////////
        Derived2 d2;
        x = d2.f();        // Redefinition => Derived2::f()
        // d2.f(s);        // Error: string version hidden
/////////////////////////////////////////////////////////////////////////////////////////////////

    class Derived3 : public Base {
    public:
        // Change return type:
        void f() const { cout << "changed returnType => void Derived3::f() const\n"; }
    };

///////////////////////////////////////// main //////////////////////////////////////////////////
        Derived3 d3;
        d3.f();            // changed returnType => void Derived3::f() const
        // x = d3.f();     // Error: return int version hidden
/////////////////////////////////////////////////////////////////////////////////////////////////

    class Derived4 : public Base {

/////////////////////////////////////////// main /////////////////////////////////////////////////

Empty Class:

If you create an object for an Empty class then the compiler will provide the default version of the following functions if they are not explicitly declared.

class A {
    A();                     // default constructor
    A(const A&);             // copy constructor
    A& operator=(const A&);  // copy assignment operator
    ~A();                    // destructor
};

An empty class has a size greater than zero. Two class objects of empty classes will have distinct addresses, as in:

class A {};

void f() {
        A* p1 = new A; // allocate a new memory
        A* p2 = new A; // allocate a new memory
        // p1 != p2 at this point ...
}

Note:

1. In C, we can't declare an empty structure but in C++ we can declare both the empty structure and class - which can be used as a skeleton or placeholder while in the initial stage of the program development.

2. In C++, like C - Suppose if we create a class with a size is 0, then two objects created one after the other would point to the same address. So having the class size as a minimum of 1 effectively ensure that the addresses of two different objects will always point to a different address.


Instance of a Class:

An instance of a class can be created by a method called a constructor. There are 5 possible ways the Constructor can be called:

1. Default Constructor 2. Copy Constructor 3. Copy Assignment operator 4. Constructor with argument and 5. Conversion Constructor.   

class A {
    A() { /* ... */ }                      // default constructor
    A(const A& a) { /* ... */ }            // copy constructor
    A& operator=(const A& a) { /* ... */ } // copy assignment
    A(int i, int j) { /* ... */ }          // constructor with argument
};

class B {
    B() { /* ... */ }           // default constructor
    B(const A& a) { /* ... */ } // conversion constructor
};

void function() {
    A a0(0, 0); // constructor with argument
    A a1(a0);   // copy constructor
    B b1(a1);   // conversion constructor
    B b;        // default constructor
    b1 = a0;    // conversion constructor
    a0 = a1;    // copy assignment
}
