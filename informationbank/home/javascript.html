Introduction:
* JavaScript was created in 1995 by Brendan Eich while he was an engineer at Netscape.
* JavaScript is a prototype-based language that doesn't have classes; instead, the class functionality is accomplished by functions and object prototypes.
* Functions are objects, giving functions the capability to hold executable code and be passed around like any other object.

1. Number: The number type represents both integer and floating-point numbers. But all numbers are implemented as IEEE 754 double-precision 64-bit floating point number (There's no dedicated data type implementation for the integer numbers). 

And it includes integer values in the range: [-9007199254740991; 9007199254740991]. 
Both values are defined as constants: Number.MIN_SAFE_INTEGER and Number.MAX_SAFE_INTEGER correspondingly.  
0.1 + 0.2 == 0.30000000000000004;
 
Special numeric values:  which also belong to this data type: Infinity, -Infinity and NaN.

Infinity - It is a special value that’s greater than any number. Note: Infinity === Infinity
    alert( 1 / 0 ); // Infinity
    alert( Infinity ); // Infinity

NaN  - represents a computational error / Not a number. It is a result of an incorrect or an undefined mathematical operation, for instance:
    alert( "not a number" / 2 );     // NaN, such division is erroneous

NaN is sticky. Any further operation on NaN returns NaN:
    alert( "not a number" / 2 + 5 ); // NaN
So, if there’s a NaN somewhere in a mathematical expression, it propagates to the whole result. 
Doing mathematical operations are safe in JavaScript. We can do anything: divide by zero, treat non-numeric strings as numbers, etc. 
The script will never stop with a fatal error (“die”). At worst, we’ll get NaN as the result.

2. String:  A string in Javascript may consist of only one character or many of them. There is no 'char' data type for handling single characters like C++.
                It is a sequences of Unicode characters (sequences of UTF-16 code units ). Each Unicode character is represented by either 1 or 2 code units. 
                Each code unit is represented by a 16-bit number.      
    'hello'.length; // 5 code units

    // Treat String like an Object: 
    'hello'.charAt(0); // "h"
    var str = 'hello, world';
    var res = str.replace('hello', 'vanakam'); // res = "vanakam, world"; str = 'hello, world' (Immutable)
    res = str.toUpperCase(); // "HELLO, WORLD"

A string in JavaScript must be surrounded by quotes. In JavaScript, there are 3 types of quotes.
    Single quotes: 'Hello'
    Double quotes: "Hello"
    Backticks: `Hello`
Double and single quotes are “simple” quotes. There’s no difference between them in JavaScript. 

Backticks are “extended functionality” quotes. They allow us to embed variables and expressions into a string by wrapping them in ${…}, for example:
    var name = "John";
    // embed a variable
    alert( `Hello, ${name}!` ); // Hello, John!
    // embed an expression
    alert( `the result is ${1 + 2}` ); // the result is 3

slice Vs substring: Both does mostly the same but it has 2 main difference.
    str.substring(beginIndex[, endIndex]);
    str.slice(beginIndex[, endIndex]);
    // both will return a new string containing the extracted section of the string.

1) Negative indexes: 
        Substring requires positive indexes and will set a negative index to 0. 
        Slice's negative index means the position from the end of the string.
    "1234".substring(-2, -1) == "1234".substring(0,0) == ""
    "1234".slice(-2, -1)     == "1234".slice(2, 3)    == "3"

2) Swapping of indexes: 
        Substring will reorder the indexes to make the first index less than or equal to the second index.
        Slice doesn't swap the indexes.
    "1234".substring(3,2) == "1234".substring(2,3) == "3"
    "1234".slice(3,2)     == ""

Examples:
    let anyString = 'Mozilla';
    // Displays 'M'
    console.log(anyString.substring(0, 1))
    console.log(anyString.substring(1, 0)); // swapping takes place
    // Displays 'lla'
    console.log(anyString.substring(4));
    console.log(anyString.substring(4, 7)); // 7-4 = 3 characters picked 
    console.log(anyString.substring(7, 4));

    // Extract last 3 characters
    const id = "0123456789";

Why 'this' value is inconsistent ?  
The most important thing to understand is that a function object does NOT have a fixed 'this' value -- the value of this changes depending on how the function is called. 
We say that a function is invoked with some a particular 'this' value -- that 'this' value is determined at invocation time, not definition time.

    - If the function is called a "raw" function (e.g., someFunc()), 'this' will be the global object (window in a browser) (or undefined if the function runs in strict mode).
    - If it is called as a method on an object, 'this' will be the calling object.
    - If it is called as an event listener, 'this' will be the element that is the target of the event.
    - If it is called as a constructor with new, 'this' will be a newly-created object whose prototype is set to the prototype property of the constructor function.
    - If you call a function with a call or apply, 'this' is specified as the first argument to call or apply.
    - If the function is the result of a bind operation, the function will always and forever have 'this' set to the first argument of the bind call that produced it. 
         (This is the single exception to the "functions don't have a fixed this" rule -- functions produced by bind actually do have an immutable 'this'.)

Using var that = this; is a way to store the 'this' value at function definition time (rather than function execution time, when this could be anything, depending on how the function was invoked). The solution here is to store the outer value of 'this' in a variable (traditionally called 'that' or 'self') which is included in the scope of the newly-defined function, because newly-defined functions have access to variables defined in their outer scope.

C++ OO vs Javascript OO:
C++ OO : In classical inheritance, the programmer writes a class, which defines an object. Multiple objects can be instantiated from the same class. In Javascript, every object has a secret link to the object which created it, forming a chain. When an object is asked for a property that it does not have, its parent object will be asked... continually up the chain until the property is found or until the root object is reached.

Not everything is an object in JavaScript:
    1. strings, numbers, boolean are primitives(), NOT objects. Unlike objects, primitive values are immutable.
    2. Primitives do have object wrappers (String, Number and Boolean); these objects have methods and properties while the primitives do not.
    3. JavaScript silently creates a wrapper object when code attempts to access any property of a primitive.
    var x = "sri"; . NOT a String type. But how it has a 'substring' method ?
    var sub = x.substring(1, 2); // sub is now have the string "r"
Behind the scenes, x.substring(1, 2) will undergo the following operations:
    1. Create a wrapper String object from x, equivalent to using new String(x)
    2. Call the substring() method with the appropriate parameters on the String object returned by step 1
    3. Dispose of the String object
    4. Return the string primitive NOT String Object from step 2.

A consequence of this is string primitive 'x'  look like a string object but really it is NOT. If you assign properties to primitives, it is pointless because you cannot retrieve them:

    x.bar = "cheese"; // x is changed to String object for accepting the property BUT it will be destroyed immediately after this line.
    alert(x.bar);  // undefined - lost!
This happens because the property is effectively defined on a String object that is immediately discarded.
