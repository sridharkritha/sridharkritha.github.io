Static Member Variable and Static Member function:

Static Member Variable: Static member variables only exist once in a program regardless of how many class objects are defined !. 
This means all objects of a class share the same static variables.

    class Something {
    public:
        static int s_nValue;
        static char Name[ ];
    };
    
    int Something::s_nValue = 1;  // If not initalized then by default it value is 0.
    char Something::Name[25] = "Sridhar Krishnan";
    
    void main() {
        Something cFirst;
        cFirst.s_nValue = 2;   // Bad practise to use the object for accessing the static variable
    
        Something cSecond;
        cout << cSecond.s_nValue;  // 2 - Bad practise

        cout<<Something::s_nValue; // 2  - Correct way of accessing without using any object.
    }

Static Member Function:
    Suppose if the static member variable is declared under the private section of a class then it can't be accessed outside the class. In this case, the static member function comes into the picture - Like static member variables, static member functions are not attached to any particular object, they can be called directly by using the class name and the scope operator. Like static member variables, they can also be called through objects of the class type, though this is not recommended. So by declaring the static member function under the public section of the class allows to access the static member variable under the private section.

Two properties of Static Member function:
    1. Static member function does not have "this" pointer because it's not attached to any object.
    2. Static member functions can only access static member variables. They can not access non-static member variables. This is because non-static
        member variables must belong to a class object, and static member functions have no class object to work with!

    class Something {
        private: static int s_nValue; // OK but NOT allowed initalization is => "static int s_nValue = 5"
                 int value = 5;       // OK allowed intitialization
        public:  static int GetValue() { return s_nValue; }
    };

    int Something::s_nValue = 5;     // OK - private memeber access allowed for setting the value
    // int Something::s_nValue = 15; // NOT allowed for second time initialization

    int main() {
        // cout << Something::s_nValue << std::endl; // NO access. s_nValue is privare. 
        cout << Something::GetValue() << std::endl;  // Prints 5
    }

Singleton:
    Singleton Design Pattern creates one single instance of a class that can be used throughout multiple files and large projects.
Think of C++ Singleton like a global class that can be accessed at any time and will be created only once.

Note: Singleton method can be used to solve the global variables used across the project. The variables that need to be accessed across different files in the projects are packed into a singleton class and then a single instance of the variables pack used across the projects.


    class Singleton {
    public:
        // Method to get the object:
        // Instantiate the class for the first time by validating the member pointer.
        // If the member pointer is already pointing to some valid memory it means that the
        // first object is already created and it should not allow for the next instantiation
        // so simply return the member pointer without calling its constructor.
        static Singleton* getInstance();

        // data to check the functionality
        int data;
    private:
        // A member pointer which point to the first created object and the same pointer will be
        // returned for any number of object created for this class (Nothing but the first instance)
        static Singleton* pInstance;

        // Default constructor, copy-constructor and assignment operators should be inside private
        // so that nobody from outside can call those functions and instantiate it
        Singleton();
        Singleton(const Singleton&);
        Singleton& operator= (const Singleton&);
    };

    // initialize pointer
    Singleton* Singleton::pInstance = NULL;

    Singleton* Singleton::getInstance()
    {
        if (pInstance == NULL) // is it the first call?
        {
            pInstance = new Singleton; // create instance - calls constructor
        }

        return pInstance; // address of first instance
    }

    Singleton::Singleton() {        
        data = 10; // perform necessary instance initializations
    }

    int main() {
        Singleton *p1 = Singleton::getInstance();
        cout<<"\n Value of date ="<< p1->data;
        p1->data = 20;

        Singleton *p2 = p1->getInstance();
        cout<<"\n Value of date ="<< p2->data;
        p2->data = 30;

        Singleton & ref = * Singleton::getInstance();
        cout<<"\n Value of date ="<< ref.data ;
    }

Create an array of objects:
    class MyClass {
        int x;
    public:
        void setX(int i) { x = i; } 
    };
    
    int main()  {
        MyClass obs[3];   // Static Array of Object
        MyClass *p = new MyClass[3]; // Dynamic Array of Object
        
        for(int i=0; i < 3; i++)
        {
            obs[i].setX(i);
            p[i].setX(i);
        }

        MyClass *q   = new MyClass ;   // Creat single dynamic object
        MyClass &ref = *(new MyClass); // dereferencing the pointer returned by new (Alternate Method)
        q->setX(5);
        ref.setX(10);
    }
