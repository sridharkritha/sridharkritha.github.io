7. Floating Point arithmetic:
    0.5 + 0.25 = 0.75 but Why 0.1 + 0.2 = 0.30000000000000004 ?

Representation of above decimal fraction is base 10: 

Base - 10 form:     a / (2n x 5m) = a / 10k


    0.5  = 5 x 10⁻¹ = 50 x 10⁻²
    0.25 = 25 x 10⁻²
    Adding above values gives 75 x 10⁻²= 0.75

    0.1  = 1 x 10⁻¹
    0.2  = 2 x 10⁻¹
    Adding above values gives 3 x 10⁻¹= 0.3 

So in decimal system(base 10) it is not a problem but computers uses binary system(base 2)

Representation of same above decimal fraction is base 2:

Base - 2 form:        a / 2n

Note: Every binary fraction can be written in decimal, the reverse is not true.

    0.5  = 1 x 2⁻¹ = 2 x 2⁻²
    0.25 = 1 x 2⁻²
    Adding above values gives3 x 2⁻²= 0.75 Not problem in base 2 system either.

But
0.1 or 0.2 or 0.3 - We cannot represent the exact value in base 2 (NOT in inverse multiple of 2). So it involves some rounding errors(approximation).
 
The problem comes with numbers that can be represented exactly in base 10, but not in base 2. Those numbers need to be rounded to their closest equivalent. 
Some of the possible fraction numbers (inverse multiple of 2) which can be represented by base 2 without any rounding issue:
 n x 2⁻¹ = n x 0.5
         = 0.5, 1.0, 1.5, 2.0, 2.5,...........etc.,

 n x 2⁻² = n x 0.25
         = 0.25, 0.5, 0.75, 1.0,...............etc., 

Note: 0.1 + 0.4 = 0.5 (gives exact value 0.5 even though both 0.1 and 0.4 can NOT be represented in base 2 exactly. 
The reason is behind the screen it involves rounding of 0.1 and 0.4, rounding of addition value to 0.5 and then 0.5 
can be exactly represented by base 2 ).

Find no. of digits in 12345 ?
floor(log₁₀(12345)) + 1 = floor(4.091) + 1 =  5 digits


Simillarly, 
    Single digit(min, max) = (1,9)
    NumberOfDigits(1) = floor(log₁₀(1)) + 1 = floor(0) + 1     = 1 digit
    NumberOfDigits(9) = floor(log₁₀(9)) + 1 = floor(0.955) + 1 =  1 digit

    Double digit(min, max) = (10, 99)
    NumberOfDigits(10) = floor(log₁₀(10)) + 1 = floor(1) + 1    = 2 digits
    NumberOfDigits(99) = floor(log₁₀(99)) + 1 = floor(1.99) + 1 = 2 digits


How to calculate the precision of a floating-point number ?

log₁₀(largest_possible_fraction)

Each bit in a float, stores 2 possibilities - a binary number of n bits can store a number up to 2ⁿ

Float uses 23 bits for significant and 1 extra leading bit for internal purpose(This is NOT a sign bit), so total number of bits is 24.
floor(log₁₀(2 ^ 24)) ≈ 7.22 ≈ 7 decimal digits of precision (Do NOT add +1 in this case)


Double uses 52 bits for significant and 1 extra leading bit for internal purpose(This is NOT a sign bit), so total number of bits is 53.
floor(log₁₀(2 ^ 53)) ≈ 15.96 ≈ 15 decimal digits of precision (Do NOT add + 1 in this case)

3. setTimeout and Loop: Closure, Callbacks, Scope
    var a = [10, 20, 30];    

    function print()    {
        for(var i = 0; i < 3; ++i)
        {
            // Proble 1: Out of range
            setTimeout(function() {
                console.log(a[i]); // Undefined Undefined Undefined bcos a[i] is ALWAYS a[3]
            }.bind(this), 100);    // regardless of with or without bind(this)

            // Problem 2: Always Last value
            var local = i;
            setTimeout(function() {
                console.log(a[local]); // 30 30 30 bcos a[local] is ALWAYS a[2]
            }.bind(this), 100);

            // Solution 1: closure by function wrap. Move the setTimout into a separate function.
            wrapper(i); // Each function will havee an unique scope

            // Solution 2: Wrap up using 1. Anonymous function and 2. Argument passing
            // (function() { })(); // Anonymous function
            (function(param) { 
                setTimeout(function() {
                    console.log(a[param]); // 10 20 30
                }.bind(this), 100);
            })(i);

            // Solution 3: IIF and Argument Passing
            setTimeout(function(k) {
                console.log(a[k]); // 10 20 30
            }.bind(this, i), 1000);
        }
    }

    function wrapper(i) {
        setTimeout(function() {
            console.log(a[i]); // 10 20 30
        }, 100);  
    }

5. Simple JavaScript Inheritance by John Resig:

/* Simple JavaScript Inheritance
 * By John Resig http://ejohn.org/
 * https://johnresig.com/blog/simple-javascript-inheritance/
 * MIT Licensed.
 */
// Inspired by base2 and Prototype
// class.js
(function(){
    var initializing = false, fnTest = /xyz/.test(function(){xyz;}) ? /\b_super\b/ : /.*/;
    // The base Class implementation (does nothing)
    this.Class = function(){};
    
    // Create a new Class that inherits from this class
    Class.extend = function(prop) {
      var _super = this.prototype;
      
      // Instantiate a base class (but only create the instance,
      // don't run the init constructor)
      initializing = true;
      var prototype = new this();
      initializing = false;
      
      // Copy the properties over onto the new prototype
      for (var name in prop) {
        // Check if we're overwriting an existing function
        prototype[name] = typeof prop[name] == "function" && 
          typeof _super[name] == "function" && fnTest.test(prop[name]) ?
          (function(name, fn){
            return function() {
              var tmp = this._super;
              
              // Add a new ._super() method that is the same method
              // but on the super-class
              this._super = _super[name];
              
              // The method only need to be bound temporarily, so we
              // remove it when we're done executing
              var ret = fn.apply(this, arguments);
              this._super = tmp;
              
              return ret;
            };
          })(name, prop[name]) :
          prop[name];
      }
      
      // The dummy class constructor
      function Class() {
        // All construction is actually done in the init method
        if ( !initializing && this.init )
          this.init.apply(this, arguments);
      }
      
      // Populate our constructed prototype object
      Class.prototype = prototype;
      
      // Enforce the constructor to be what we expect
      Class.prototype.constructor = Class;
  
      // And make this class extendable
      Class.extend = arguments.callee;
      
      return Class;
    };
  })();
