Note: Both 'prototype' and '__proto__' are properties (typeof this property is an object).

Definition 1:
    'prototype' created automatically when declaring class/ function /  Function Object / Constructor Function.
    '__proto__' created explicitly by creating an 'instance' out of class/function/constructor function using 'new'.

    ( new Foo ).__proto__ === Foo.prototype;
    ( new Foo ).prototype === undefined; // instance do NOT have 'prototype' property

    function Person(name) { this.name = name }; 
    var eve = new Person("Eve");
    eve.__proto__ == Person.prototype  //true - NOT recommended
Object.getPrototypeOf(eve) == Person.prototype //true same as above - Recommended

    eve.prototype  //undefined
Note: Instances have __proto__, classes have prototype.

Definition 2:
    The __proto__ property of an object is a property that maps to the prototype of the constructor function of the object. 
In other words:
    instance.__proto__ === constructor.prototype // true
This is used to form the prototype chain of an object.

Ex 1: To understand this let us create a function
    function Foo(name) {
        this.name = name;
    }

When JavaScript executes this code, it adds 'prototype' property to Foo, prototype property is an object with 2 properties in it:
    constructor
    __proto__

So when we do, Foo.prototype it returns
    constructor: Foo  // function definition
    __proto__: Object
Now as you can see constructor is nothing but the function itself and __proto__ points to the root level Object of JavaScript.

Let us see what happens when we use a function with new key word.
    var b = new Foo('JavaScript');
When JavaScript executes this code it does 4 things:
    1. It creates a new object, an empty object // {}
    2. It creates __proto__ on b and makes it point to Foo.prototype so b.__proto__ === Foo.prototype
    3. It executes Foo.prototype.constructor (which is the definition of function Foo ) with the newly created object (created in step#1) as 
        its context (this), hence the name property passed as 'JavaScript' (which is added to this) gets added to the newly created object.
    4. It returns the newly created object in (created in step#1) so var b gets assigned to the newly created object.

Now if we add Foo.prototype.car = "BMW" and do b.car, the output "BMW" appears. This is because when JavaScript executed this code it searched for car property on b, it did not find then JavaScript use b.__proto__ (which was made to point to 'Foo.prototype' in step#2) and finds car property so return "BMW".

Example 2:
    function Foo () { }

    // prototype ( object property) was created automatically when we declared the function Foo.
    Foo.hasOwnProperty('prototype'); // true

    // Now, we can assign additional properties and methods to it. And which will be shared by all the Foo 
// instances created using new Foo().
    Foo.prototype.name = 'sridhar';
    Foo.prototype.myName = function () { return 'My name is ' + this.name; }
Every instance created using new Foo() has a __proto__ property which points to the Foo.prototype. 

If you create a new object out of Foo using the new keyword, you are basically creating a new object that has an internal(private) link to the function Foo's prototype.
    var b = new Foo();
    var c = new Foo();
    b.__proto__ === Foo.prototype // true
    c.__proto__ === Foo.prototype // true
Both __proto__ of b and c points to the same Foo.prototype. But __proto__ of b.__proto__ is unique for b and __proto__ of c.__proto__ is unique for c

Any object can have access to all the properties in its proto chain as linked by __proto__ , thus forming the basis for prototypal inheritance.
    b.[[Prototype]] === Foo.prototype   // true
    b.__proto__ === Foo.prototype  // true
    Object.getPrototypeOf(b) === b.__proto__ // true

1. Object Prototype:
    var obj = {}; // (same as) var obj = new Object();
    console.log(obj.__proto__ === Object.prototype); // '__proto__' property created for 'obj'
    console.log(obj); // [object Object] - how this string is printed even though obj is empty and no toString method has defined ?
                      // Answer: toString method is inherited from the build-in Object()

    console.log(obj.toString === obj.__proto__.toString == Object.prototype.toString); // true
    console.log(Object.prototype.__proto__); // null -> ROOT - No more chain

Similarly, all built-in objects such as Array, Date, Function have Object.prototype on the top.
    var arr = [1, 2, 3];
    // it inherits from Array.prototype?
    alert( arr.__proto__ === Array.prototype ); // true
    // then from Object.prototype?
    alert( arr.__proto__.__proto__ === Object.prototype ); // true
    // and null on the top.
    alert( arr.__proto__.__proto__.__proto__ ); // null -> ROOT - No more chain

Some methods in prototypes may overlap, for instance, Array.prototype has its own toString that lists comma-delimited elements:
    var arr = [1, 2, 3]
    alert(arr); // 1,2,3 <-- the result of Array.prototype.toString

Primitives data types (string, number, bool): 
They are not objects. But if we try to access by property, then temporary wrapper objects are created using built-in constructors String, Number, Boolean, they provide the methods and disappear. Their corresponding prototypes are String.prototype, Number.prototype and Boolean.prototype.

null and undefined: They have no object wrappers, no properties and no prototypes.

Changing native prototypes (Not recommended): Native prototypes can be modified. For instance, if we add a method to String.prototype, it becomes available to all strings
    String.prototype.show = function() {
        alert(this);
    };
    "BOOM!".show(); // BOOM!

Borrowing from prototypes: Some methods of native prototypes are often borrowed. For instance, if we’re making an array-like object, we may want to copy some Array methods to it. E.g.
     // Below obj to borrow the join() method from an Array
    var obj = { 0: "Hello",
                  1: "world!",
                 length: 2
   };
    obj.join = Array.prototype.join; // borrowing only ONE method
    // (or) if obj is NOT inherits from any another object then you can do below "inheritance"
    obj.__proto__ = Array.prototype; // borrowing ALL the methods from Array

    alert( obj.join(',') ); // Hello,world!
It works, because the internal algorithm of the built-in join method only cares about the correct indexes and the length property, it doesn’t check that the object is indeed the array. And many built-in methods are like that.

Thief() and Person() relationship - Inheritance:
    function Thief() { }
    var p = new Person();
    Thief.prototype = p; // this determines the prototype for any new Thief objects: "Inheritance"
    var t = new Thief();

Prototype chain of  instance/object 't' is one step longer than that of instance/object 'p' :
    t → p → Person.prototype → Object.prototype (end point)

The other Prototype chain of class/function Person is not longer: Thief and Person are siblings sharing the same parent in their prototype chain:
    Person / Thief → Function.prototype → Object.prototype (end point)

Note: Object's  '__proto__ ' property and  Function 'prototype' property are different but it links together.

* Only 'function' has a 'prototype' property.
    obj.__proto__ = aFunction.prototype

* Objects created with 'new' are linked to the prototype property of their 'constructor':
    var Fun = function() {}; // Some function

If we use a 'new' operator before any function then we call it as a constructor function [ new Fun() ].  
If an Object is created by a constructor [ new fun() ] then the object get a __proto__ property.
    var obj = new Fun(); // internally will do => obj.__proto__ === Fun.prototype

* Just assume, constructor function is a class which has a 'prototype object' has a property. That 'prototype object' can have properties which will be shared across the difference instances of the class.
* If a function is never used as a constructor, its prototype property will go unused.
* If you don't need a constructor, use 'Object.create' instead of 'new'.
    var obj = Object.create(anotherObject); // internally will do => obj.__proto__ = anotherObject

* If you look up a property via obj.propName or obj['propName'] -  the runtime looks up the property in the object referenced by __proto__ instead. 
* Every 'object' has an internal 'prototype' property and it links to another object. Some browsers allow direct access to this property via a non-standard property named __proto__ . But the Standard way is Object.getPrototypeOf().

Shallow Cloning: Object.create performs an object cloning more powerful than copying properties using  for..in:
    // fully identical shallow clone of obj
    var clone = Object.create(Object.getPrototypeOf(obj), Object.getOwnPropertyDescriptors(obj));
This call makes a truly exact copy of obj, including all properties: enumerable and non-enumerable, data properties and setters/getters – everything, and with the right [[Prototype]].

Plain Object: 
     1. All JavaScript object inherits from Object by default, unless you explicitly create it with null as its prototype, like: Object.create(null).
     2. Even the empty object {}  will inherits from Object and equivalent to Object.create(Object.prototype). 
         So Object.create(null) has NO __proto__ property, while {} does.

Plain Object is a good choice for creating plain hash map storage because it doesn't require testing hasOwnProperty() among their property entries.

An object which has no __proto__  means __proto__ can also be used just like a property key string. 
    var obj = Object.create(null); // creates an empty object without a prototype
    obj.__proto__ = 'sridhar';
    console.log(obj.__proto__); // sridhar
    console.log(obj); // Error (NO toString method)

Constructor and Prototype:
    var Dog = function () {}; // the constructor function
    Dog.prototype.bark = function () {}; // adding a method to Dog.prototype
    var dog1 = new Dog; // creating a new instance. Internally it will do linkage: dog1.__proto__ = Dog.prototype
    dog1.bark(); // the instance inherits the "bark" method from Dog.prototype


1. In JavaScript, mostly everything is an object. Primitive types are NOT object but they will be wrapped as an object at the run time when there is a need.
2. All objects contain [[Prototype]] internal property (or) __proto__. It links to another object. 

        Ex1: If it is a non functional object then linkage will be
    obj [[Prototype]] -> anotherObject
        [or]
    obj.__proto__ === anotherObject

        Ex 2: If it is a function object then linkage will be
    obj [[Prototype]] -> func.prototype
            [or]
    obj.__proto__ === func.prototype
        Ex: dog1 object's [[Prototype]] refers to Dog.prototype.  
3. Only function objects have built-in .prototype property - not every object. That's the reason only functional objects can only be used as 
    a constructor.
4. [[Prototype]] and .prototype are not the same.
5. [[Prototype]] link only happen at the time of object creation by 'new' / Object.create(someObject).
    // Method 1: new
    function A() {}
    var obj = new A(); // obj.__proto__ = A.prototype

    // Method 2: Object.create
    var obj = Object.create(anotherObject); // obj.__proto__ = anotherObject

6. Objects created with new are linked to the prototype property of their own constructor.
7. If you don't need a constructor, use Object.create instead of new.

For example, take a simple array: var a = []. You can make operations with it, like a.push(10). Where does this push method come from? From the prototype of Array object. You can add your own methods to Array class/function just by defining them in the prototype object. For example:
    Array.prototype.sortNum = function() { this.sort(function(a, b) { return a - b; });};
This way you can do something like a.sortNum(). If you want to define your own classes, you'll have to define its prototype to define the methods for all the instances of that class:

    function Class() {}; // Constructor function
    Class.prototype.foo = function() { alert("foo"); };
    Class.salary = 65000; // will NOT be inherited
    Class.prototype.age = 12; // inherited

    function Subclass() {};
    Subclass.prototype = new Class(); // inheritance: adding one more level in the path
    Subclass.prototype.bar = function() { alert("bar"); };

    var a = new Class(); // a.__proto__ === Class.prototype
    var b = new Subclass();
    alert(b.salary); //undefined
    alert(b.age); // 12
    a.foo(); // alerts"foo"
    // a.bar(); // throws an Error: a.bar is not a function
    b.foo(); // alerts "foo"
    b.bar(); // alerts "bar"

    a instanceof Class; // true
    a instanceof Subclass; // false
    b instanceof Class; // true

Two models for implementing classes and instances in JavaScript: 
                                1. Closure way
                                2. Prototyping (Inheritance) way
1. Closure way:
    function Shape(x, y) {
        var that = this; // must - local variable -> private and also a closure
        this.x = x; this.y = y;
        this.toString = function() { return 'Shape at '+ that.x +', '+ that.y; };
    }

    function Circle(x, y, r) {
        var that = this; // must

        Shape.call(this, x, y);
        this.r = r;

// NOT inherited, every function/class has in-build toString method access
        var _baseToString = this.toString; // must
        this.toString = function() {
            return 'Circular ' + _baseToString(that) + ' with radius '+that.r;
        };
    }

    var c = new Circle(1,2,3);
    console.log('' + c); // "Circular Shape at 1, 2 with radius 3"

Disadvantage:
    1. Now every single instance of Shape / Circle will have its own copy of the toString method
    2. instanceof wouldn't work

Advantage:
    1. Binding is come for free bcos each instances owns separate copy of the methods.

    alert(temp()); // "Circular Shape at 1, 2 with radius 3"

2. Prototyping way (Inheritance):
Basics:
    // some independent functions Parent & Child
    function Parent(name)  { this.name = name; }
    function Child(street) { this.street = street; }

    // Method 1: Inheritance - Old way
    Child.prototype = new Parent(name); // Parent constructor is executed internally

    // Method 2: Inheritance - ES5 way
    Child.prototype = Object.create(Parent.prototype); // Parent constructor is NOT executed
    Child.prototype.constructor = Child; // reset back to Child from Parent

To make the Parent constructor to execute while doing the inheritance, by adding a explicit call to Parent from Child.
    function Child(street) {
        Parent(street); // passing some arguments. This is optional
        this.street = street;
    }

If you want to pass ALL arguments of Child to Parent, you can also use Function.prototype.apply() :
    function Child(street) {
        Parent.apply(this, arguments); // passing ALL arguments. This is optional
        this.street = street;
    }

    function Circle(x, y, r) {

        this.r= r;
    }

    // Method 1: old way

Disadvantage:

                                name: this.name, // Parent
                                address: this.street, // Child
                                year: this.year // GrandChild
                           };
    };

'this' and Scope:

Exactly 5 different ways in which the value of 'this' can be bound in the Javascript.

1. Global Scope:
    this; // When using 'this' in global scope, it will simply refer to the global object. 

2. Calling a Function:
    foo(); // Here, 'this' will again refer to the global object.

ES5 Note: In strict mode, the global/window value is no longer exists for 'this'. 'this' will instead have the value of 'undefined' in that case.

3. Calling a Method:
    test.foo(); // this will now refer to 'test'.

4. Calling a Constructor
    new foo();  // 1. A function call that is preceded by the new keyword acts as a constructor. 
                // 2. Inside the function, this will refer to a newly created Object.

5. Explicit setting of 'this':
    function foo(a, b, c) { }

    var bar = {};
    foo.apply(bar, [1, 2, 3]); // this inside the called function gets explicitly set to the first argument of 'apply'
    foo.call(bar, 1, 2, 3);    // this inside the called function gets explicitly set to the first argument of 'call'


Common misconception about 'this': 
1. Function inside a method
Problem:
    Foo.method = function() {
        function test() {
            // this = global object NOT Foo object
        }
        test();
    }

Solution: In order to gain access to Foo from within test, you can create a local variable inside of method that refers to Foo.
Method 1:
    Foo.method = function() {
        var self = this;
        function test() {
            // self = Foo object (closure helps to access) and do NOT use 'this' here
        }
        test();
    }

Method 2:
    Foo.method = function() {
        var test = function() {
            // this now refers to Foo object
        }.bind(this);
        test();
    }

Relationship between window vs self vs this :
       * Unless set elsewhere, the value of 'self' is 'window' because JavaScript lets you access any property x of window as simply x, instead of window.x. 
       * Therefore, 'self' is really 'window.self', which is different to 'this'.

If you're using a function that is executed in the global scope and is not in strict mode, 'this' defaults to window, and therefore
    function foo() {
        console.log(
            window.self === window, // is self window?
            window.self === this,   // is self this?
            this === window         // is this window?
        );
    }
    foo(); // true true true

If you're using a function in a different context, 'this' will refer to that context, but 'self' will still be window.
    // invoke foo with context {}
    foo.call({}); // true false false

Find below some of the combination of 'window', 'self' and 'this' console output's in global context on the browser environment.
    console.log( window ); // Window {…}
    console.log( self );   // Window {…}
    console.log( this );   // Window {…}

    console.log( window.window.window );    // Window {…} - recursive self reference
    console.log( self.self.self );          // Window {…} - recursive self reference
    console.log( window.self.window.self ); // Window {…} - recursive self reference
    console.log( self.window.self.window ); // Window {…} - recursive self reference
    console.log( this.this );               // undefined

    console.log( window.window ); // Window {…}
    console.log( window.self );   // Window {…}
    console.log( window.this );   // undefined  

    console.log( self.self );     // Window {…}
    console.log( self.window );   // Window {…}
    console.log( self.this );     // undefined

    console.log( this.this );     // undefined
    console.log( this.window );   // Window {…} - important
    console.log( this.self );     // Window {…}

2. Assigning Methods to a variable: Method aliasing does NOT work in JavaScript.
    var test = someObject.methodTest; // Method aliasing (NOT a function aliasing)
    test(); // test now acts like a plain function call; therefore, 'this' inside it will NO longer refer to 'someObject'.
Note: As with any function(NOT method) in JavaScript, the scope is inherited from where it is defined, not where it is invoked.

Anti- Patterns as follows:
1. Set the Prototype :
    // Title: Classical Pattern #1 - The Default Pattern (a pattern that should be generally AVOIDED)
    // Description: create an object using the Parent() constructor and assign this object to the Child()'s prototype
    function inherit(C, P) { C.prototype = new P(); }

    // the parent constructor
    function Parent(name) { this.name = name || 'Adam'; }
    // adding functionality to the prototype
    Parent.prototype.say = function () { return this.name; };

    // empty child constructor
    function Child(name) { }

    // inheritance magic happens here
    inherit(Child, Parent);

    var kid = new Child();
    console.log(kid.say()); // "Adam"

    // Drawback 1: it doesn't enable you to pass parameters to the child constructor
    var newlyBorn = new Child('Sridhar');
    console.log(newlyBorn.say()); // "Adam"

    // Drawback 2: own properties added to `this` is inherited
    var kiddo = new Child();
    kiddo.name = "Patrick";
    console.log(kiddo.say()); // "Patrick"

2. Rent a Constructor:
    // Title: Classical Pattern #2 - Rent a Constructor (a pattern that should be generally AVOIDED)
    // Description: it borrows the parent constructor, passing the child object to be bound to this and 
// also forwarding any arguments

    // the parent constructor
    function Parent(name) { this.name = name || 'Adam'; }
    // adding functionality to the prototype
    Parent.prototype.say = function () { return this.name; };

    // child constructor
    function Child(name) {
        Parent.apply(this, arguments); // Borrows the parent constructor
    }

    var kid = new Child("Patrick");
    console.log(kid.name); // "Patrick"

    // Drawback 1: nothing from the prototype gets inherited
    console.log(typeof kid.say); // "undefined"

3. Rent the multiple constructors:
    // Multiple Inheritance by Borrowing Constructor(s)
    function Cat() {
        this.legs = 4;
        this.say = function () { return "meaowww"; }; // member function NOT a prototype function
    }

    function Bird() {
        this.wings = 2;
        this.fly = true;
    }

    function CatWings() {
        Cat.apply(this);  // Borrows the parent constructor - Cat
        Bird.apply(this); // Borrows the parent constructor - Bird
    }

    var jane = new CatWings();
    console.log(jane); // { legs: 4, say: [Function (anonymous)], wings: 2, fly: true }

 4. Rent and Set the Prototype :
    // Title: Classical Pattern #3 - Rent and Set Prototype (a pattern that should be generally AVOIDED)
    // Description: first borrow the constructor and then also set the child's prototype to point to 
// a new instance of the constructor

    // the parent constructor
    function Parent(name) { this.name = name || 'Adam'; }
    // adding functionality to the prototype
    Parent.prototype.say = function () { return this.name; };

    // child constructor
    function Child(name) {
        Parent.apply(this, arguments); // Borrows the parent constructor
    }

    Child.prototype = new Parent(); // Set Prototype

    var kid = new Child("Patrick");
    console.log(kid.name);  // "Patrick"
    console.log(kid.say()); // Patrick
    console.dir(kid);       // { name: 'Patrick' }
    delete kid.name;
    console.log(kid.say()); // "Adam"

    // Drawback - the parent constructor is called twice, so it could be inefficient

5. Temporary Constructor:
    // Title: Classical Pattern - A Temporary Constructor (a pattern that should be generally AVOIDED)
    // Description: first borrow the constructor and then also set the child's prototype to point to 
// a new instance of the constructor

    // Inheritance - Basic
    function inherit(C, P) {
        // Basic Inheritance
        var F = function () { };
        F.prototype = P.prototype;
        C.prototype = new F();
        // Advanced Inheritance
        C.uber = P.prototype;         // Storing the Super class (Super = uber)
        C.prototype.constructor = C;  // Resetting the Constructor
    }
    // Note: uber method (sugar method named by Douglas Crockford) is a helper method to access the parent implementations. 

    // Inheritance using closure
    var inherit = (function () {
                    var F = function () { };
                    return function (C, P) {
                                                F.prototype = P.prototype;
                                                C.prototype = new F();
                                                C.uber = P.prototype;
                                                C.prototype.constructor = C;
                                            };
                  }());

    // the parent constructor
    function Parent(name) { this.name = name || 'Adam'; }
    // adding functionality to the prototype
    Parent.prototype.say = function () { return this.name; };

    // child constructor
    function Child(name) { }

    inherit(Child, Parent);

    var kid = new Child();
    console.log(kid.name);       // undefined
    console.log(typeof kid.say); // function
    kid.name = 'Patrick';
    console.log(kid.say());            // Patrick
    console.log(kid.constructor.name); // Child
    console.log(kid.constructor === Parent); // false

'3' level inheritance using Temporary Constructor:
    // inheritance helper
    function extend(Child, Parent) {
        var TF = function () {}; // Bcos of temporary constructor TF() - properties of PARENT class will NOT changed 
                                // ever if you change the properties of child(subclasses)    
        Child.prototype = new TF();
        Child.prototype.constructor = Child;
        Child.uber = Parent.prototype;
    }

    // define class -> define prototype variable
    function Shape() { } // 1. define
    Shape.prototype.name = 'Shape'; // 2. define prototype variable
    Shape.prototype.toString = function () {
        return this.constructor.uber ? this.constructor.uber.toString() + ', ' + this.name : this.name;
    };

    // define class -> inherit -> define prototype variable
    function TwoDShape() { }  // 1. define
    extend(TwoDShape, Shape); // 2. inherit
    TwoDShape.prototype.name = '2D shape'; // 3. define prototype variable

    // define class -> inherit -> define prototype variable
    function Triangle() { }      // 1. define
    extend(Triangle, TwoDShape); // 2. inherit
    Triangle.prototype.name = 'Triangle'; // 3. define prototype variable

    var my = new Triangle();
    console.log(my.toString()); // Shape, 2D shape, Triangle

Alternate way:
    function creature() {}
    creature.prototype.name = 'Creature';
    creature.prototype.showName = function () {
        return this.constructor.uber ? this.constructor.uber.showName() +
        ',' + this.name : this.name;
    };

    function dog() {}

    dog.prototype.constructor = dog;
    dog.prototype.name = 'Charles';
    dog.uber = creature.prototype;
    var cat = new dog();
    console.log(cat.showName()); // Creature,Charles

Recommended Patterns as follows:
Prototypal Inheritance:
    //  Description: objects inherit from other objects
    function createObjectFromAnotherObject(o) {
        function F() { }
        F.prototype = o;
        return new F();
    }

    // 1. Plain object to inherit from
    var plainObject = { name:"I'm plain Object" };

    // the new object
    var child = createObjectFromAnotherObject(plainObject);

    // testing
    console.log(child.name); // I'm plain Object

// 2. constructor function to inherit from
    function Parent() {     
        this.name = "I'm Parent"; // an "own" property
    }
    // A property added to the prototype
    Parent.prototype.getName = function () { return this.name; };

    // create a new person object
    var father = new Parent();

    // inherit
    var kid = newObject(father);

    // test that both the own property and the prototype property were inherited
    console.log(kid.getName()); // "Adam"

    // wrong way of inherit
    var kid = newObject(Person.prototype); // WRONG - Need object NOT prototype
    console.log(typeof kid.getName);       // OK - "function", because it was in the prototype
    console.log(typeof kid.name);          // "undefined", because only the prototype was inherited

    // Addition to ECMAScript 5 
    var child = Object.create(parent); // No need for user defined "createObjectFromAnotherObject(o) { }"

    var child = Object.create(parent, {
        age:{ value:2 } // ECMA5 descriptor
    });
    console.log(child.hasOwnProperty("age")); // true

Inheritance by Copying Properties :
    // Description: an object gets functionality from another object, simply by copying it

    // Shallow copy
    function extend(parent, child) {
        var i;
        child = child || {};
        for (i in parent) {
            if (parent.hasOwnProperty(i)) {
                child[i] = parent[i];
            }
        }
        return child;
    }

    // Deep copy
    function extendDeep(parent, child) {
        var i;
        child = child || {};
        for (i in parent) {
            if (parent.hasOwnProperty(i)) {
                if (typeof parent[i] === 'object') {
                    child[i] = (Object.prototype.toString.call(parent[i]) === "[object Array]") ? [] : {};
                    extendDeep(parent[i], child[i]); // Recursive call
                } else {
                    child[i] = parent[i];
                }
            }
        }
        return child;
    }

    var dad = {
                    counts:[1, 2, 3],
                    reads:{paper:true}
                };

    // Deep Copy - Each will have it's own copies
    var kid = extendDeep(dad);
    kid.counts.push(4);
    console.log(kid.counts.toString());   // "1,2,3,4"
    console.log(dad.counts.toString());   // "1,2,3"
    console.log(dad.reads === kid.reads); // false - 2 Object NEVER be equal
    console.log(kid.reads);               // { paper: true }

    // Shallow Copy - Shared Memory
    var child = extend(dad);
    child.counts.push(4);
    console.log(dad.counts.toString());     // "1,2,3,4"
    console.log(dad.reads === child.reads); // true

Mix-ins:
    // Description: copy from any number of objects and mix them all into a new object
    function mix() {
        var arg, prop, child = {};
        for (arg = 0; arg < arguments.length; arg += 1) {
            for (prop in arguments[arg]) {
                if (arguments[arg].hasOwnProperty(prop)) {
                    child[prop] = arguments[arg][prop];
                }
            }
        }
        return child;
    }

    var cake = mix( {eggs:2, large:true},
                    {butter:1, salted:true},
                    {flour:'3 cups'},
                    {sugar:'sure!'}
                 );

    console.log(cake);

Borrowing Methods:
    // Description: reuse one or two methods of an existing object without forming a parent-child relationship 
// with that object.
    var objectOne = {   name:'objectOneName',
                        say:function (greet) { return greet + ', ' + this.name; }
                    };

    var objectTwo = { name:'objectTwoName' };

    // apply two to original object one.
    console.log(objectOne.say.apply(objectTwo, ['hello'])); // hello, objectTwoName

    // assigning to some variable, `this` will point to the global object
    var say = objectOne.say;
    console.log(say('hoho')); // "hoho, undefined"

    function bind(obj, mtd) { // obj, mtd forms closure
        return function () {
            return mtd.apply(obj, [].slice.call(arguments));// objectOne.say.apply(objectTwo, 'Hai')
        };
    }

    var twosay = bind(objectTwo, objectOne.say); // returns function with closure. Also Borrows a function from another object.
    console.log(twosay('Hai'));       // Hai, objectTwoName

    // ECMAScript 5 adds a method bind() to Function.prototype, making it just as easy to use as apply() and call().
    if (typeof Function.prototype.bind === 'undefined') {
        Function.prototype.bind = function (thisArg) {
            var fn = this,
                    slice = Array.prototype.slice,
                    args = slice.call(arguments, 1); // delete the 1st argument
            return function () {
                    return fn.apply(thisArg, args.concat(slice.call(arguments))); // convert to array and apply
            };
        };
    }

    var twosay = objectOne.say.bind(objectTwo); // bind(objectTwo, objectOne.say);
    console.log(twosay('Hai')); // Hai, objectTwoName
