Properties and Methods of HTML element
Navigating Between Nodes: The following 6 node properties allow you to navigate the DOM tree using JavaScript.  
1. parentNode    
                2. childNodes[nodeNumber]
                3. firstChild
                3. lastChild
                4. nextSibling
                5. previousSibling

DOM navigation properties are great when elements are close to each other. What if they are not? How to get an arbitrary element of the page?
There are another 6 main methods to search for nodes in DOM:

Besides that:
    1. There is elem.matches(css) to check if elem matches the given CSS selector.
    2. There is elem.closest(css) to look for the nearest ancestor that matches the given CSS-selector. The elem itself is also checked.
And let’s mention one more method here to check for the child-parent relationship, as it’s sometimes useful:
    3. elemA.contains(elemB) returns true if elemB is inside elemA (a descendant of elemA) or when elemA==elemB.

Note: getElement* always should only be used with document (document.getElementsByName()) but querySelectorall, querySelctor, matches. closest can be applied to both document and element (document.querySelector(css) and elem.querySelector(css)).

elem.querySelectorAll(css): The querySelectorAll method uses CSS selector syntax to find elements. You can match classes by preceding the class name with a dot, and IDs by using a pound-sign (#). Elements can be matched by just using the name of the tag.

    var classEle = document.querySelectorAll('.treehouse');
    var idEle = document.querySelectorAll('#treehouse');
    var tagEle = document.querySelectorAll('div');

You can combine multiple selectors to create more complex queries just as you would in CSS.

    var element = document.querySelectorAll('#main.treehouse');

Print all <li> elements which are the last child of <ul> element:

    <ul>
        <li>Muthu</li>
        <li>Sridhar</li>
    </ul>
    <ul>
        <li>Kavitha</li>
        <li>Vijaya</li>
    </ul>

    <script>
        let elements = document.querySelectorAll('ul > li:last-child');
        for (let elem of elements) {
            alert(elem.innerHTML); // "Sridhar", "Vijaya". Note: .innerHTML is used
        }
    </script>

Note: Pseudo-classes in the CSS selector like :hover and :active are also supported. For instance, document.querySelectorAll(':hover') will return the collection with elements that the pointer is over now.

elem.querySelector(css): The querySelectorAll method will return a list of all the elements that match your query. If you just wish to return a single element you can use the querySelector method. This will return the first element that matches the query.

elem.querySelector(css) returns the first element for the given CSS selector. That result is the same as elem.querySelectorAll(css)[0], but the latter is looking for all elements and picking one, while elem.querySelector just looks for one. So it’s faster and also shorter to write.

elem.match(css): Return true/false if match finds the given css-selector.

    <a href="http://example.com/file.zip">...</a>
    <a href="http://ya.ru">...</a>
    <script>
        // can be any collection instead of document.body.children
        for (let elem of document.body.children) {
            if (elem.matches('a[href$="zip"]')) {
                alert("The archive reference: " + elem.href);  // The archive reference: http://example.com/file.zip
            }
        }
    </script>

elem.closest(css): Returns closest ancestor(parent or grandparent NOT children's) element.

    <h1>Contents</h1>
    <div class="contents">
        <ul class="book">
            <li class="chapter">Chapter 1</li>
            <li class="chapter">Chapter 1</li>
        </ul>
    </div>
    <script>
        let chapter = document.querySelector('.chapter'); // LI
        alert(chapter.closest('.book')); // UL
        alert(chapter.closest('.contents')); // DIV
        alert(chapter.closest('h1')); // null (because h1 is not an ancestor)
    </script>

Live collections: All methods "getElementsBy*" return a live collection. Such collections always reflect the current state of the document and “auto-update” when it changes.

    <div>First div</div>
    <script>
        let divs_getEls = document.getElementsByTagName('div');
        let divs_qrySel = document.querySelectorAll('div');
        alert(divs_getEls.length); // 1
        alert(divs_qrySel.length); // 1
    </script>
    <div>Second div</div>
    <script>
        alert(divs_getEls.length); // 2
        alert(divs_qrySel.length); // 1 - NOT auto updated
    </script>

document.getElementById or id: If an element has the id attribute, we can get the element using the method document.getElementById(id), no matter where it is.

    <div id="parent">
        <div id="childOne">One</div>
        <div id="child-two">Two</div>
    </div>

    <script>
        let elem = document.getElementById('parent'); // get the element
        elem.style.background = 'red'; // make its background red
        // childOne is a reference to DOM-element with id="childOne"
        childOne.style.background = 'red'; // (NOT recommended) You can directly access 'id' even WITHOUT document.getElementById
        child-two.style.background = 'red'; // ERROR. id="child-two" has a hyphen inside, so it can't be a variable name
        window['child-two'].style.background = 'red'; // OK (direct access NOT recommended)
    </script>

'nodeValue'  or  'innerHTML'  property: Expecting an element node to contain plain text is a frequent error. Actually it contains a node (text node )which contains text. using the 'nodeValue' property, or 'innerHTML' can access the text from the text node which is attached internally with the element node.

    <html>
        <body>
            <div id="el1">Amazingly Creative Heading</div>
            <div id="el2">Look this text DOM!</div>
            <script>
                document.getElementById("el2").innerHTML = document.getElementById("el1").childNodes[0].nodeValue;
                                            // or
                document.getElementById("el2").innerHTML = document.getElementById("el1").firstChild.nodeValue;
                                            // or
                document.getElementById("el2").innerHTML = document.getElementById("el1").innerHTML;
            </script>
        </body>
    </html>


// Output:
            Amazingly Creative Title
            Amazingly Creative Title

nodeValue property is used to specify the value of a node. It has 3 possible return values:

  1. NULL (in case of a document or element node)
  2. Content (in case of a text or comment node)
  3. Attribute value (in case of an attribute node)

The nodeType property is used to return the type of node.

NodeList HTMLCollection

Contains all types of nodes (element node, attribute node, text node,comment node) Contains only element/tag nodes(<div>,<body>, etc)

Items ONLY accessed by their index number Items can be accessed by their index number, name, or id

Mostly Static collection. But in some cases it is live collection as well.
Ex: elem.childNodes => live NodeList		 Live collection  - automatically updated when the underlying document is changed

    <div id=”container”>
        <div class="divy">...</div>
        <div class="divy">...</div>
    </div>

    <script>
        var htmlColl = document.getElementsByClassName("divy")
        htmlColl.constructor.name; //=> HTMLCollection
        var nodeColl = document.querySelectorAll(".divy");
        nodeColl.constructor.name; //=> NodeList
        htmlColl.length; // 2
        nodeColl.length; // 2
        //append new child to container
        var newDiv = document.createElement('div');
        newDiv.className = 'divy';
        document.getElementById('container').appendChild(newDiv);
        htmlColl.length; // 3 (live update)
        nodeColl.length; // 2
    </script>


    // Examples 
    document.getElementsByTagName => (live) HTMLCollection
    element.children              => (live) HTMLCollection
    element.childNodes            => (live)   NodeList
    document.querySelectorAll     => (static) NodeList


Both collections have a length property like array but it doesn't support array methods like push(), pop(), or join().

Convert the array like collection into real array using splice:


    Array.isArray(htmlColl);   //=> false
    var divyArray = Array.prototype.slice.call(htmlColl); // converted to Array
    Array.isArray(nodeColl);   //=> false
    var divyArray = Array.prototype.slice.call(nodeColl); // converted to Array

Browser parses/converts 'HTML elements' to 'DOM objects'

  1. HTML Element 'attributes' are mapped to DOM object 'properties'
  2. If HTML Element is <body id="page">, then the DOM object has body.id="page".
  3. HTML element attributes are case-insensitive. (id == iD)
  4. DOM object properties are case-sensitive. (id != iD)
A small comparison:

Attributes – is what’s written in HTML.

Properties – is what’s in DOM objects.

Attribute        Property

   id                     id

  name            name

  class            className

  for               htmlFor

Properties Attributes
Type Any value, standard properties have types described in the spec A string
Name Name is case-sensitive Name is not case-sensitive

Methods to work with attributes are:

elem.hasAttribute(name)               – to check for existence.

elem.getAttribute(name)                – to get the value.

elem.setAttribute(name, value)      – to set the value.

elem.removeAttribute(name)         – to remove the attribute.

elem.attributes                                - is a collection of all attributes.

For most situations using DOM properties is preferable. We should refer to attributes only when DOM properties do not suit us, when we need exactly attributes, for instance:

  1. We need a non-standard attribute. But if it starts with data-, then we should use dataset.
  2. We want to read the value “as written” in HTML. The value of the DOM property may be different, for instance the href property is always a full URL, and we may want to get the “original” value.


Reading the standard and non-standard attributes from HTML elements:

HTML:

     <!-- Attributes name is case in-sensitive and also ALWAYS STRING -->
    <body ID="STANDARD" something="non-standard"> </body>

Javascript:

    // Standard Attribute Names
    console.log(document.body.iD); // STANDARD
    console.log(document.body.id); // STANDARD    
    // Non-Standard attribute does not yield a property
    console.log(document.body.something); // undefined: DOM property NOT created, because it's non-standard
    // 'getAttribute' to get the non-standard HTML attribute from the HTML element
    console.log(document.body.getAttribute('something')); // non-standard    
    // 'setAttribute' to add a new non-standard attribute to the HTML element
     document.body.setAttribute('Test', 123); // 123 becomes "123" string value. Test = "123"    
    // .outerHTML - shows all attributes attached to the HTML element
    alert( document.body.outerHTML ); // (3), see if the attribute is in HTML (yes)    
    // 'attributes' to read ALL attribues from an Element.
    // The attributes collection is iterable and has all the attributes of the element (standard and non-standard) 
    // as objects with 'name' and 'value' properties.
    for (let attr of document.body.attributes) {
      console.log( `${attr.name} = ${attr.value}` );   // id = STANDARD
    }                                                  // something = non-standard
                                                       // test = 123

DOM 'Property' - HTML 'Attribute' synchronization:

Standard Attribute: Changing DOM property will change HTML attibute. Also changing HTML attribute will change DOM property.

Non-Standard Attribute: Those attributes are not as a DOM element property. You need to directly change in the HTML element using 'setAttribute'.

Exception in the synchronization: Standard HTML 'vALue' attribute and DOM '.value' property will follow only one way synchronization from HTML -> DOM NOT from DOM -> HTML

HTML:

    <body>
        <input>
    </body>

Javascript:

    let input = document.querySelector('input');
    // attribute => property
    input.setAttribute('iD', 'inputId'); // Directly update the HTML Element attribute 
    console.log(input.id);               // DOM property 'id' automatically updates to "inputId"
    // property => attribute
    input.id = 'newId';                    // Change DOM property 'id' value
    console.log(input.getAttribute('Id')); // HTML Element attribute value automatically update to "newId"
    // Exceptional 'value' attribute
    // attribute => property
    input.setAttribute('value', 'Hello world');
    console.log(input.value); // 'Hello world'
    // NOT property => attribute
    input.value = 'Sridhar Krishan';
    console.log(input.getAttribute('value')); // 'Hello world' (NOT updated to 'Sridhar Krishan'!)

Examples:

    <input id="the-input" type="text" value="Name:">

id: The id property is a PURE reflected property for the id attribute. Changing one will reflect other.

type: The type property is a reflected property but NOT PURE of the type attribute.  Changing one will reflect other. But the changes uses standard type name. Otherwise it wouldn't reflect.

    <input type="foo">
    theInput.getAttribute("type"); // "foo"
    theInput.type; // "text".

value: the value property doesn't reflect the value attribute. The value property will reflect the current value in the text box. But value attribute will have the initial value.

    theInput.value;                  // returns "John"
    theInput.getAttribute('value');  // returns "Name:"

Other exceptions:

  1. checked - 'key' only boolean attribute (value is always true when it is present in the HTML element)  

    <input id="input" type="checkbox" checked>

 

    alert(input.getAttribute('checked')); // the attribute value is: "empty string"
    alert(input.checked);                 // the property value is: true


2. style - attirbute is a String. style - property is an Object

    <div id="div" style="color:red;font-size:120%">Hello</div>


    // string
    alert(div.getAttribute('style')); // color:red;font-size:120%
    // object
    alert(div.style);                // [object CSSStyleDeclaration]
    alert(div.style.color);          // red


3. href - DOM property is always a full URL, even if the attribute contains a relative URL or just a #hash.

    <a id="a" href="#hello">link</a>


    // attribute
    alert(a.getAttribute('href')); // #hello
    // property
    alert(a.href );               // full URL in the form http://site.com/page#hello


Non-standard attributes: Non-standard attributes are used to pass custom data from HTML to JavaScript, or to “mark” HTML- elements for JavaScript.

    <!-- mark the div to show "name" here -->
    <div show-info="name"></div>
    <!-- and age here -->
    <div show-info="age"></div>


    // the code finds an element with the mark and shows what's requested
    let user = {
        name: "Pete",
        age: 25
    };
    for(let div of document.querySelectorAll('[show-info]')) {
        // insert the corresponding info into the field
        let field = div.getAttribute('show-info');
        div.innerHTML = user[field]; // first Pete into "name", then 25 into "age"
    }

Dataset: What if we use a non-standard attribute for our purposes and later the standard introduces it and makes it do something?. To avoid conflicts, there exist data-* attributes. All attributes starting with “data-” are reserved for programmers’ use. They are available in the dataset property.

For instance, if an elem has an attribute named "data-about", it’s available as elem.dataset.about.

Multiword attributes like data-order-state become camel-cased: dataset.orderState.

Ex 1:

    <body data-about="Elephants">
        <script>
        alert(document.body.dataset.about); // Elephants
        </script>
    </body>


Ex 2:

HTML:

  <div id="order" class="order-colour" data-order-state="new"> my colour </div>

CSS:

    .order-colour[data-order-state="new"]      { color: green; }
    .order-colour[data-order-state="pending"]  { color: blue;  }
    .order-colour[data-order-state="canceled"] { color: red;   }

Note: attribute be preferable to having classes like .order-state-new, .order-state-pending, order-state-canceled ?. Because an attribute is more convenient to manage than adding/removing css class.

Javascript:

    // On loading - Green
    console.log(order.dataset.orderState); // my colour  (Green at beginning)
    // modify - Blue
    order.dataset.orderState = "pending"; // my colour  (changed to Blue)
