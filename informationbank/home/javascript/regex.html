    word1 && word2 && word3

    (?=.*word1)(?=.*word2)(?=.*word3)
    The .* in the first lookahead lets it match however many characters it needs to before it gets to "word1". 
    Then the match position is reset and the second lookahead seeks out "word2". Reset again, and the final part 
    matches "word3"; since it's the last word you're checking for, it isn't necessary that it be in a lookahead, 
    but it doesn't hurt.

How to create the Regular expressions ? There are 2 syntax's to create a regular expression object.
var regexp = /pattern/flags;

var regexp = new RegExp("pattern", "flags");

Advantages of using 'new RegExp' over / /
    -  allows to construct a pattern dynamically from a string
    -  support variable insertions ${...}

Example:
    var tag = "h2";
    var regexp = new RegExp(`<${tag}>`);    
    alert( "<h1> <h2> <h3>".search(regexp)); // 5 - start index of <h2>

Flags: Only 6 flags (gym is u) are supported in Javascript.
    g - look for all getMatchedCSSRules, without it - only the first one
    y - StickY mode
m - multiline mode
  i - ignore case difference (A === a)    
    s - 'Dotall' mode, allows . to match newlines
u - Enables unicode support    

search(substr|regexp): method looks for the pattern(/love/) or substring ('love') and returns the position inside the string or -1 if there’s no match.
    var str = "I love JavaScript!";
var substr = 'love'; // Javascript way
    alert( str.search(substr) ); // 2

    var regexp = /Love/i; // Regex way:
    alert( str.search(regexp) ); // 2
Note: flag 'g' is useless for the search method because it only finds the first match. We can’t find next matches using search, there’s just no syntax for that.

match(regexp): 
    str.match(reg) (no - g flag) - Array of matches. Arr[0] - full first match and Arr[1...n] other properties.
    str.match(reg) (g flag)         - Array of all matches and NO other properties and don't create any elements for parentheses.
If there are no matches, the result is null, NOT an empty array.

Note: If a part of the pattern is delimited by parentheses (...), then it becomes a separate element in the array.

no - g flag:
    var str = "JavaScript is a programming language";
    var result = str.match( /JAVA(SCRIPT)/i );
    alert( result[0] ); // JavaScript (the whole match)
    alert( result[1] ); // script (the part of the match that corresponds to the parentheses)
    alert( result.index ); // 0
    alert( result.input ); // JavaScript is a programming language

g flag:
    var str = "HO-Ho-ho!";
    var result = str.match( /h(o)/ig );
    alert( result ); // HO, Ho, ho


split(str|regexp, limit): Splits the string using the regexp (or a substring) as a delimiter.
    var a = '12-34-56'.split('-'); // array of [12, 34, 56]

    var b = '12-34-56'.split(/-/); // array of [12, 34, 56]

replace(str|regexp, str|func): The swiss army knife for search and replace.
    // replace only first dash by a colon
    alert('12-34-56'.replace("-", ":")); // 12:34-56
    alert('12-34-56'.replace(/-/, ":")); // 12:34-56

    // replace ALL dashes by a colon
    alert('12-34-56'.replace(/-/g, ":")) // 12:34:56

Note: special characters and callback function can also be used in the second argument of a replacement string.

A. special characters Usage:
    $&      the whole match
    $`      a part of the string BEFORE the match
    $'      a part of the string AFTER the match
    $n      $1, $2 (and so on). if n is a 1-2 digit number, then it means the contents of n-th parentheses counting from left to right,
            otherwise it means a parentheses with the given name

Examples:
    // prepend all entries of "John" with "Mr.":
    var str = "John Doe, John Smith and John Bull";
    alert(str.replace(/John/g, 'Mr.$&')); // Mr.John Doe, Mr.John Smith and Mr.John Bull

Swap first and last name : To do so, we should:
           1. First, mark the parts by parentheses in regexp.
           2. Use $1, $2 (and so on) in the replacement string to get the content matched by 1st, 2nd and then swap it.
    var str = "John Smith";
    alert(str.replace(/(john) (smith)/i, '$2, $1')) // Smith, John

B. callback function(str, p1, p2, ..., pn, offset, input, groups) usage: It will be called for each match and its result will be inserted as a replacement.

5 Optional Arguments:
    str – the match,
    p1, p2, ..., pn – contents of parentheses (if there are any),
    offset – position of the match,
    input – the source string,
    groups – an object with named groups (see chapter Capturing groups).

A. 5 arguments: In the example below there are 2 parentheses, so replacer is called with 5 arguments: str is the full match, then content of parentheses, and then offset and input:
    function replacer(str, name, surname, offset, input) {
    // name is the first parentheses, surname is the second one
    return surname + ", " + name;
    }

    var str = "John Smith";
    console.log(str.replace(/(John) (Smith)/, replacer)); // Smith, John

B. 3 arguments: If there are no parentheses in the regexp, then there are only 3 arguments: function(str, offset, input).
    // show and replace all matches
    function replacer(str, offset, input) {
        console.log(`Found ${str} at position ${offset} in string ${input}`);
        return str.toLowerCase();
    }
    
    var result = "HO-Ho-ho".replace(/ho/gi, replacer);
    console.log( 'Result: ' + result ); // Result: ho-ho-ho
    // Found HO at position 0 in string HO-Ho-ho
    // Found Ho at position 3 in string HO-Ho-ho
    // Found ho at position 6 in string HO-Ho-ho

C. No arguments:
    var i = 0;
    // replace each "ho" by the result of the function
    alert("HO-Ho-ho".replace(/ho/gi, function() { return ++i; })); // 1-2-3


regexp.exec(str):  It behaves differently depending on whether the regexp has the g flag or not.
 1. no 'g' used in 'regexp.exec' then
   regexp.exec(str) === str.match(regexp)

 2. 'g' used in 'regexp.exec' then
1. regexp.exec(str) returns the first match and remembers the position after it in regexp.lastIndex property.
2. The next call starts to search from regexp.lastIndex and returns the next match.
3. If there are no more matches then regexp.exec returns null and regexp.lastIndex is set to 0.
    var str = 'A lot about JavaScript at https://javascript.info';
    var regexp = /javascript/ig;    
    var result;
    
    while (result = regexp.exec(str)) {
     console.log( `Found ${result[0]} at ${result.index}` );
     // Found JavaScript at 12
     // Found javascript at 34
    }

Also we can start search from given index position.
    regexp.lastIndex = 13;
    while (result = regexp.exec(str)) {
        console.log( `Found ${result[0]} at ${result.index}` ); 
        // Found javascript at 34
    }

regexp.test(str): looks for a match and returns true/false whether it finds it. It behaves differently depending on whether the regexp has the g flag or not.
 1. no 'g' used in 'regexp.test' then
    /regexp/.test(str) === (str.search(regexp) !== -1)

2. 'g' is used then regexp.test advances regexp.lastIndex property, just like regexp.exec.
var regexp = /love/gi;
      var str = "I love JavaScript";
      regexp.lastIndex = 10; // start the search from position 10:
      alert( regexp.test(str) ); // false (no match)
 
Hint:
        var regexp = /javascript/g; // (regexp just created: regexp.lastIndex=0)
        alert( regexp.test("javascript") ); // true (regexp.lastIndex=10 now)
        alert( regexp.test("javascript") ); // false
To work around that, one could use non-global regexps or re-adjust regexp.lastIndex = 0 before a new search.

Character classes:
    \d = [0-9] - any ONE digit ( 0 to 9). NOTE: [1-9] is NOT equal to \d
    \s = [\t\n\v\f\r ] - space (spaces, tabs, newlines, other unicode space characters)
    \w = [a-zA-Z0-9_] - any ONE character (A English character, a single digit and an underscore - Non English single character
are not allowed). Ex: \d\s\w matches '9 p'
    \b - word boundary

Inverse classes: For every character 'classes' there exists an “inverse class”, denoted with the same letter, but uppercase.
    \D = [^\d] - any ONE character except \d. Ex: P
    \S = [^\s] - any ONE character except \s. Ex: 9
    \W = [^\w] - any ONE character except \w. Ex:
    \B = [^\b] - anything but \b. Ex:

Convert the given phone number "+7(903)-123-45-67" to pure number "79035419441"
     var str = "+7(903)-123-45-67";
     console.log( str.match(/\d/) ); // 7

     console.log( str.match(/\d/g) ); // array of matches: 7,9,0,3,1,2,3,4,5,6,7

     console.log( str.match(/\d/g).join('') ); // 79035419441

     // Using inverse classes
     console.log( str.replace(/\D/g, "") ); // 79031234567
 
Any word character:
    console.log( "I love HTML5!".match(/\s\w\w\w\w\d/) ); // 'HTML5'    

    console.log( "twenty third".match(/\w/) ); // t ["t", index: 0, input: "twenty third", groups: undefined]
    console.log( "twenty third".match(/\w+/) ); // twenty
    console.log( "twentythird".match(/\w+/) ); // twentythird
    console.log( "twenty-third".match(/\w+/) ); // twenty
    console.log( "twenty third".match(/\w+\s\w+/) ); // twenty third
    console.log( "twenty-third".match(/\w+-\w+/) ); // twenty-third

Word boundary (\b): Force where in the string the match should take place.  
        * It checks the boundaries(beginning or end) of a word.
        * It is 'zero width' and doesn't physically matches any character
        * More or less similar like the markers ^ (beginning of a line) and $ (end of a line). [ cap to dollar => head to foot ]
    console.log( "dogname is tommy".match(/^dog/) ); // dog
    console.log( "tommy is thedog".match(/dog$/) ); // dog
    console.log( "dog".match(/^dog$/) ); // dog ["dog", index: 0, input: "dog", groups: undefined]

    console.log( "mydogname is tommy".match(/dog/) ); // dog    
    console.log( "mydog name is tommy".match(/\bdog/) ); // null
    console.log( "my dogname is tommy".match(/\bdog/) ); // dog

Quantifiers +, *, ? and {n} :

Quantity {n} : A quantifier is appended to a character class, or a [...] set and specifies how many we need.

Two modes of operations supported by Quantifiers: Greedy and Lazy modes.
Greedy Mode (default mode): Regular expression engine tries to repeat the quantifier as many times as possible. For instance, \d+ consumes all possible digits. When it becomes impossible to consume more (no more digits or string end), then it continues to match the rest of the pattern. If there’s no match then it decreases the number of repetitions (backtracks) and tries again.

Lazy Mode: Enabled by the question mark(?) after the quantifier. The regexp engine tries to match the rest of the pattern before each repetition of the quantifier.

Problem 1: Match all the sub-strings only inside the double quotes

Greedy Approach:
    // Wrong - what we expected is - witch, broom
    console.log( 'a "witch" and her "broom" is one'.match(/".+"/g)); // "witch" and her "broom"

The match looks like this:
    Step 1: ( ".+ ) Match the quote and all characters till nothing to match
        ".............................|(till end of string)
        "witch" and her "broom" is one
    Step 2: (") Decreases the number of repetitions (backtracks) and match the rest of the pattern
        "....................." <-----|
        "witch" and her "broom" is one

Convert Greedy to Lazy mode by '?'
Adding '?' after the quantifiers(by default Greedy mode) becomes Lazy mode quantifier, so that it becomes *? or +? or even ??

Lazy Approach:
    // Correct
    console.log( 'a "witch" and her "broom" is one'.match(/".+?"/g)); // witch, broom

Alternate solution without Lazy mode using [^]+:
    // Correct
    console.log( 'a "witch" and her "broom" is one'.match(/"[^"]+"/g)); // witch, broom

"[^"]+" looks for a quote(") followed by one or more non-quotes [^"], and then it stops the repetitions when it meets the closing quote.

Problem 2: Find all Html statement of the form <a href="......" class="doc">

Greedy Approach:

    // Wrong! Two links in one match!
    console.log( str.match(/<a href=".*" class="doc">/g) ); // <a href="link1" class="doc">... <a href="link2" class="doc">

The match looks like this:

    <a href="link1" class="doc">... <a href="link2" class="doc"></a>

Lazy Approach:

    // Works!
    console.log( str.match(/<a href=".*?" class="doc">/g) ); // <a href="link1" class="doc">, <a href="link2" class="doc"></a>

Now it seems to work, there are two matches:

    <a href="link1" class="doc">... <a href="link2" class="doc"></a>

Lookahead, Lookbehind and Lookaround:
    look    : ?
    ahead   : =
    behind  : >
    around  : ( ( ) )

Lookahead x(?=y)  Suppose we need to match a pattern only if it is followed(after) by an another pattern.
    x(?=y), it means "look for x, but match only if followed by y".

Extract the number before €
    var str = "Purchasing 2 turkey costs 30€";
    console.log( str.match(/\d+(?=€)/) ); // 30 (correctly skipped the sole number 2)

NOT Lookahead x(?!y) 
x(?!y) "search x, but only if not followed by y".

Extract the number which is NOT before €
    var str = "Purchasing 2 turkey costs 30€";
    console.log( str.match(/\d+(?!€)/) );// 2 (correctly skipped the price)

Note: ECMASCRIPT 5 doesn't support Lookbehind
Lookbehind (?<=y)x  Suppose we need to match a pattern only if it is precede(before) by an another pattern.
    (?<=y)x, it means "matches x, but only if it follows after y".

Extract the number after $
    var str = "Purchasing 2 turkey costs $50";
    console.log( str.match(/(?<=\$)\d+/) ); // 50 (correctly skipped the sole number 2)

NOT Lookbehind (?<!y)x
    (?<!y)x, it means " matches x, but only if there’s no y before".

Extract the number which is NOT after $
    var str = "Purchasing 2 turkey costs $50";
    console.log( str.match(/(?<!\$)\d+/) ); // 2 (correctly skipped the price)

Lookaround: ( both lookahead and lookbehind) and Capture groups used by lookaround:
In general both lookahead and lookbehind don't include the match inside the parentheses (€ $ matches in the above examples). But if you wrap into another parentheses then you can capture the match inside the inner parentheses.

    var str = "Purchasing 2 turkey costs 30Rs";
    var reg = /\d+(?=(€|Rs|\$))/; // extra parentheses around €|Rs|\$
    console.log( str.match(reg) ); // 30, Rs

    var str = "Purchasing 2 turkey costs $30";
    var reg = /(?<=(\$|£))\d+/;
    console.log( str.match(reg) ); // 30, $

[\s\S]*  match any character including line break. [+ve,-ve] (popular combination)
[\s\S\r]* match any character including line break. [+ve,-ve, carriage return] (Windows needs additionally '\r' for line break)
[\w\W]*   match any character including line break. [+ve,-ve]
[^\r]*    match any character including line break. [+ve,-ve, carriage return]
[^]       Only works in JS.
Note: \r is NOT needed for Unix flavour OS

`hello zebra, how are you ?
hello pARROT, how are you ?`.replace(/z[^]*?p/,'sridhar'); // "hello sridharARROT, how are you ?"

Any text between the two closest 'z' and 'p'

.+ (. means any NON newline character and + means one or more)
[] denotes a character class and will therefore only ever match ONE character. 


'Greedy' means match longest possible string.
'Lazy'  means match shortest possible string.

For example, the greedy h.+l matches 'hell' in 'hello' but the lazy h.+?l matches 'hel'.

to match a line that doesn't contain a word(sridhar)

/^((?!sridhar).)*$/.test("hello sridhar krishnan") // false  - contains sridhar
/^((?!sri).)*$/.test("hello sridhar krishnan")  // false  - contains sridhar
/^((?!sridhav).)*$/.test("hello sridhar")       // true - doesn't contains sridhar


^ - the beginning of the string, 
(  - group and capture to \1 (0 or more times (matching the most amount possible)),
(?!  - look ahead to see if there is not,
sridhar - your string,
)  - end of look-ahead, 
.  - any character except \n,
)*  - end of \1 (Note: because you are using a quantifier on this capture, only the LAST repetition of the captured pattern 
will be stored in \1)
$  - before an optional \n, and the end of the string


Regex for password must contain 
at least eight characters, 
at least one number and 
both lower and uppercase letters and 
special characters

You may use this regex with multiple lookahead assertions [AND (&&)conditions]:

^(?=.*?[A-Z])(?=.*?[a-z])(?=.*?[0-9])(?=.*?[#?!@$%^&*-]).{8,}$
This regex will enforce these rules:

At least one upper case English letter, (?=.*?[A-Z])
At least one lower case English letter, (?=.*?[a-z])
At least one digit, (?=.*?[0-9])
At least one special character, (?=.*?[#?!@$%^&*-])
Minimum eight in length .{8,} (with the anchors)

The following are metacharacters (, ), [, ], {, }, ., ?, +, *. They have a specific meaning when appearing in a pattern.
 To match them literally they must be backslash-escaped. To match a backslash literally, backslash-escape it: \\.

 Metacharacters: .*+\-?^${}()|[\]\\
They have a specific meaning when appearing in a pattern.
 To match them literally they must be backslash-escaped. Ex: To match a backslash literally, backslash-escape it: \\.
 If it is present inside the string then they must be backslash-escaped as well. 

 Write a function which escape the metacharacters in the given string:
 function escapeRegExp(string) {
    return string.replace(/[.*+\-?^${}()|[\]\\]/g, '\\$&'); // $& means the whole matched string
  }
