mongoDB







MongoDB:  It is document(json) database. [db -> collection -> Documents]

  Install MongoDB server from https://www.mongodb.com/
  Test the MongoDB is working or not  by typing in cmd: mongod

  Ref: The Magical Marvels of MonogoDB by Joel Taylor (Pluralsight)

  1. NoSQL database
  2. Collection instead of Tables
  3. Document instead of ROW. So Collection is Nothing but collection of standalone documents(Not like One table is related to other by Relation schema in SQL).
      DB -> Collection -> Document
  4. Documents are stored in a format called BSON (JSON + ISO DATE support+ some extra type support). Max each doc size is 16MB.
  5. You can write a Javascript program directly to MongoDB CLI.
  
  1. Create a databse using cli:
    > use studentDB          
  2. show the available databases:
    > dbs
  3. insert() method from collection helps to save the document in the collection.
    > db.students.insert( { "name": "sridhar", "course": "computing"} )
 
    Note: We can have document (embedded) inside an another document object.

    > db.students.insert( { "name": "sridhar", "fee": 20000, "marks": [98, 85, 70], "course": "computing", "mustCourse":"math", "otherCourses": ["English", "Tamil", "Hindi"],
              "address": {"area": "Chennai", "country": "India" }} )


    collection => students, if not exist then create it automatically
    document   => { "name": "sridhar", "course": "computing"}
    Note: MongoDb automatically adds '_id: some unique object id number' inside every document 

  4. remove the document from the collection
    >db.students.remove({"name": "Sridhar"})
  4. find() method from collection helps to get ALL the document from the collection
    >db.students.find()
  5. find(query) the get specific document(s) from the collection
    >db.students.find({"course": "computing"})
  6. find entries from the embedded document.
    >db.students.find({"address.country": "India"})
  7. find(multiple queries)
    >db.students.find({"course": "computing", "address.country": "India"})
  8. Query using comparision operators:
  $gt, $gte, $lt, $lte, $ne
  greater than, greater than equal to,.....
  >db.students.find({"fee": {"$lt": 2000}})
  >db.students.find({"fee": {"$lt": 2000, "$gt": 50000}}) 
 
  >db.students.find({"marks": {"$lt": 2000, "$gt": 50000}}) // Both condition should match
   // Atleast one condition to Match - use "$elemMatch"
  >db.students.find({"marks": {"$elemMatch":{"$lt": 2000, "$gt": 50000}}}) // searching inside an array by "$elemMatch"
  atlease 1 value in array MUST be greather than 50000 and less than 2000

  9. projection - extract the fields we want back by setting their value to true.
  >db.students.find({"fee": {"$lt": 2000}}, {"name": true, "course": true});
  result => {"name": "sridhar", "course":"computing"}

  Note:   >db.students.find({"fee": {"$lt": 2000}}, {"name": true, "course": false}) // ERROR - You can't do true/false selection at the same time


  10. Excludes the fields we don't want
  >db.students.find({"fee": {"$lt": 2000}}, {"name": false, "course": false});
  result => { , "fee": 20000, "marks": [98, 85, 70], "mustCourse":"math", "otherCourses": ["English", "Tamil", "Hindi"],
  "address": {"area": "Chennai", "country": "India" }}
  11. Exclude the automatically generated "_id"
  db.students.find({"fee": {"$lt": 2000}}, {name": true, "_id": false}); // no Error only for _id you can do true/false selection at the same time.

  Cursor object: Whenever we search for doc, an object is returned from the find() method called "cursor object". By default, first 20 doc are printed out.
  Using cursor object you can traverse on search method.

  Cursor methods:
  count(): db.student.find().count() // count no.of docs in the collection. 
  sort() : db.student.find().sort({"fee": 1}) // sort by fee in ascending order(1), (-1) - descending order
  skip() and limit(): db.student.find().skip(0).limit(3) // don't skip any result documents at the front and show result from first 3 result document.
  db.student.find().skip(4).limit(3) // skip first 4 result documents at the front and show result from next 3 result document.

  Avoid Duplicates by document reference using "_id": 

  > db.students.insert({ "_id": "AnnaUniversity", "name": "sridhar", "course": "computing"}) // AnnaUniversity details are referred from the   below documents
  > db.students.insert({ "UniversityDetails": "AnnaUniversity", "area": "Guindy", "State": "TamilNadu"})

  Note: MongoDB doen't support multi document update in one go unlike releational databse which update the referenced tables by one go (termed as transaction)

  Embedding Vs Referencing:
  Using single query you can change all the fields.
  Needs 2 separate queries for changing the fields.



  7. update the document: Change course from 'computing' to 'physics'
    >db.student.update({"name": "sridhar"}, {"$set": {"course": "physics"}})
                      >>>> Query Param  <<<<        >>> Update Param <<<<
    result:
    { "name": "sridhar", "course": "physics", "address": {"area": "Chennai", "country": "India" }

    Without using update operator "$set":

    >db.student.update({"name": "sridhar"}, {"course": "physics"})

    result:
    {"course": "physics"}

    Above method is useful for import data.

    Note: Update only applies to first matching document.
  8. To apply the update on all matching documents NOT just the first matching document.
    >db.student.update({"mustCourse":"math"}, {"$set": {"course": "physics"}}, {"multi": true})

  9. '$inc' operator can be used to increment a value in a document. Useful for recording the view count.
  log document in the logs collection.
    >db.logs.update("name": "sridhar", {"$inc": {"count": 1}})
    count value gets automatically incremented whenever it is accessed.
  upsert: Update the existing document or creates a new one.
  >db.logs.update("name": "sridhar", {"$inc": {"count": 1}}, {"upsert": true})

  10. '$unset' operator can be used to remove specified fields.
    >db.student.update({}, {"$unset": {"course": ""}}, {"multi": true})
  above removes 'course' field in all documents in the collection.

  11. "$rename" opeator for rename the field
    >db.student.update({}, {"$rename": {"course": "majorCourse"}}, {"multi": true})

  12. "$set" operator will update value of a particular field.
  >db.student.update("name": "sridhar", {"$set": {"otherCourses.2": "French"}})

otherCourses.2 => 2 is array index
  Result:
  { "name": "sridhar", "course": "computing", "mustCourse":"math", "otherCourses": ["English", "Tamil", "French"],
              "address": {"area": "Chennai", "country": "India" }

  13. Find and replace - Without using index. 

  >db.student.update("otherCourses": "Hindi", {"$set": {"otherCourses.$": "French"}}, {"multi": true})
  instead of index use '$' - All first occurance of 'Hindi' changed to 'French'

  14. Updating the embeded document entry.

  >db.student.update("name": "sridhar", {"$set": {"address.area": "Tnagar"}})

  14. Othe possible update operators:
  $max, $min, $mul,  $push, $addToSet, $pop, $pull

  $addToSet - Just like $push adds an element to an array at the end but only if there is no duplicate.

  15. Remove first/last element from an ARRAY:

  >db.student.update("name": "sridhar", {"$pop": {"otherCourses": 1}})
  {"otherCourses": 1}  : Removes FIRST element from an Array
  {"otherCourses": -1} : Removes LAST element from an Array

  15. Remove ALL the matching element from an ARRAY:

  >db.student.update("name": "sridhar", {"$pull": {"otherCourses": "hindi"}})
