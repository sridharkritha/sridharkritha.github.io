Javascript Event Loop:






Stack: Callback related functions like setTimeout/XMLHttpRequest etc are moved from stack to Web APIs (outside of Javascript Runtime).
Web APIs: Once time elapsed incase of setTimeout or got the response for the request incase of XMLHttpRequest then it is moved from WebAPIs to Event Queue.
EventQueue: Waits for the stack to be fully EMPTY. Once it is empty it moves one cb()'s to the stack for the execution. Again once stack if fully empty then next cb() is moved from Event loop to stake. This will continue until all cb()'s are dequeued and executed at stake. Once both the stack and event queue are empty the program will terminate.

Web APIs or Browser :
They are not part of the JavaScript language itself, rather they are for external source for example from browser / nodejs.

Ex:

    const first  = () => console.log("First");
    const second = () => setTimeout(() => console.log("Second"), 500);
    const third  = () => console.log("Third");

    second();
    first();
    third();

    // Output:
    First
    Second
    Third

Node.js Architecture:



V8 - Google's open source JavaScript engine that resides in Chrome/Chromium browsers. Instead of interpreting JavaScript code on the fly like typical web browsers do, V8 translates your JS code into machine code so that it's blazing fast. V8 is written in C++. 

libuv - It provides asynchronous I/O that includes asynchronous TCP & UDP sockets, (famous) event loop, asynchronous DNS resolution, file system read/write, and etc. libuv is written in C. 

Other Low-Level Components - such as c-ares, http parser, OpenSSL, zlib, and et., These dependencies provide low-level interactions with servers to establish important functionalities such as networking, encrypting, etc.

Bindings - How can JavaScript communicate with the code written in C/C++? For this Node.js uses binding. A binding basically is a wrapper around a library written in one language and expose the library to codes written in another language so that codes written in different languages can communicate.

Application - here is your code, modules, and Node.js' built in modules, written in JavaScript.


setTimeout vs setImmediate vs process.nextTick :

setTimeout(() => { console.log("hello"); }, 0);
setImmediate(() => { console.log("hello"); }); 
process.nextTick(() => { console.log("hello"); }); 

Use setImmediate      : if you want to queue the function behind whatever I/O event callbacks that are already in the event queue. 
Use process.nextTick : to effectively queue the function at the head of the event queue so that it executes immediately after the stack is fully empty. Not recommended especially for recursive cases.


nvm:

Difference between npm and nvm?
nvm (node version manger):  Is a tool that allows you to download and install Node.js. npm comes with Node.js.
nvm -v

nvm allows you to have different version of node.js versions. If you want you can switch from one nodejs version to other.

npm (Node Package Manager): Is a tool that allows you to install javascript packages. 
npm -v

List the different versions of nodejs installed in your system:
nvm list

Ex:
    6.9.5
  * 0.12.2 (Currently using 64-bit executable)
  
Switch from one node version to another node version(6.9.5)  
  nvm use 6.9.5
  
npm:

1. Uninstall
npm uninstall request --no-save
2. Global installation of 'request' package using npm

npm install -g request --save-dev
3. List all the installed 'lib / packages' globally (without dependencies by --depth 0)

npm list -g --depth 0
4. npm root -g
show the global installation root regardless of current working directory.
ex: C:\Users\skrishnan\AppData\Roaming\nvm\v6.9.5\node_modules
5. npm root
show the directory of your npm packages
ex: C:\Users\skrishnan\AppData\Roaming\npm\node_modules
6. Project Setup:
mkdir TestProject
cd TestProject
npm init --yes    
[creats a default package.json]
npm install request --save 
[Inside your project folder installs required npm module and save it as a project dependency (it will appear in package.json)]
TestProject/node_modules/request
TestProject/package.json
TestProject/test.js
Run your program as
node test.js

Notes:
package.json:
    It gets updated whenever their is a new package installation in your project directory. So there is NO need to share the huge 'node_modules' folder to others who want to work on your project.
    'npm install' - in their end will grab the all packages with correct version from internet.


Adding external ".js" file to your project:

Old way 1:
    index.html:
        <script scr="moment.min.js"></script>
app.js:
        console.log(moment().getCurrentTime());

Old way 2:
            npm install moment --save
    index.html:
            <script scr="node_modules/moment/min/moment.min.js"></script>
    app.js:
            console.log(moment().getCurrentTime());

Modern Way: (NO need to add the url in html script tag)
            npm install moment --save

app.js:
            var moment = require('moment');
            console.log(moment().getCurrentTime());

webpack:
    require() in the "var moment = require('moment')" works well in the server side bcos it has file access permission but it don't work in browser and throw "require is NOT defined" error.
    webpack is a module bundler which fetches all the files which are mentioned in the require() and bundle into a single which browser can understand.

    > npm install webpack webpack-cli --save-dev

            install 2 packages: webpack and webpack-cli. --save-dev: save it as a development dependency (packages you need in your development environment but not on your production server.)

    > ./node_modules/.bin/webpack app.js --mode-development

            --mode-development : readable mode useful for debugging
            --mode-production  : minified mode useful for production

    Outputs single file: dist/main.js 

    Note: Each time you change the 'app.js' you need to generate a new 'dist/main.js' using webpack.

    index.html:
        <script scr="dist/main.js"></script>

Automate the above build process: Just change the package.json:

{
  "name": "modern-javascript-example",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "build": "webpack --progress --mode=production",
    "watch": "webpack --progress --watch"
  },

  "dependencies": {
    "moment": "^2.22.2"
  },
  "devDependencies": {
    "webpack": "^4.17.1",
    "webpack-cli": "^3.1.0"
  }
}

    > npm run build
                This will run webpack using configuration from the 'webpack.config.js' with the --progress option to show the % progress and the --mode=production option to minimize the code for production.
    > npm run watch
                This uses the --watch option instead to automatically re-run webpack each time any JavaScript file changes, which is great for development.

C:\Users\username\AppData\Roaming\npm\node_modules\http-server

Kill the server running on particular port:

Ctrl + C

(or)

For Windows users use the command prompt (cmd):

  1. Find the process-id which uses the particular port number (say 8484)

    netstat  -ano  |  findstr  8484

    Result: TCP 0.0.0.0:8484 0.0.0.0:0 LISTENING 21816

  2. Kill the process using the found process-id (say 21816)

    taskkill /F /PID 21816
