1. typedef Usage: 

    typedef int myinteger;
    typedef char *mystring;
    typedef void (*myfunc)();

    myinteger i;   // is equivalent to   int i;
    mystring s;    // is equivalent to   char *s;
    myfunc f;      // is equivalent to    void (*f)();

Advanced:

    void (*p[10]) (void (*)() );

       - p is an "array of 10 pointers to a function returning void and taking a pointer to another function that returns void and takes no arguments".

you can simplify it considerably by using two steps of typedef declarations.

1. Declare a typedef for "pointer to a function returning void and taking no arguments" as follows:

  typedef void (*pfv)();

2. Next, declare another typedef for "pointer to a function returning void and taking a pfv" based on step 1.

    typedef void (*pf_taking_pfv) (pfv);

Now declaring an array of 10 such pointers will be:

    pf_taking_pfv p[10];

3. Taking the Address of a Member Function

    class a;

    typedef void(a::*func)();

    class a {
        protected:
            int x;
        public:
            func f;
            void fg() {
            f=as;  // Error in C++, but it is acceptable if as() is a static function
            };
            void as();
    };

According to standard C++, the name of a member function isn't implicitly converted to its address. In this regard, member functions and ordinary functions differ. Try to change the definition of fg() to the following:

    void fg() {
        f = &a::as;
    };

4. Static Memory or Heap?
    class C {
        public:
            void staticMemory();
            void heapMemory();
    };
    void C::staticMemory() {
        int ar[100]; // always on the stack
    }

    void C::heapMemory() {
        int *p = new int[100]; // always on the heap
    }

    C  * pc = new C; // obj is created in heap memory but ar allocated on the stack

    C ac; // obj is created in stack memory but p allocated on the heap

In other words, it doesn't matter how you allocate your object; what matters is how you create the variables within the member function.

Array of Objects and Array of pointers to objects:

    class student {
        private:
            char m_name[25];
            int m_mark;
        public:
            int m_public;
            void setData(int mark, char *name )
            {
                strcpy(m_name, name);
                m_mark = mark;
            }
            void showData()
            {
                cout<< m_name << m_mark ;
            }
    };

    int main() {
        student S[10];  // Array of 10 stack objects
        for(int i=0; i<10; i++) S[i].setData(100,"SameName")

        student *p[10]; // Array of pointer to dynamic objects
        for(int i=0; i<10; i++) {
            p[i] = new student;
            p[i]->setData(100,"SameName");
        }
    }
Note: Array of pointers to objects need not necessarily reside contiguously in memory, as in the case of a static array of object.

Accessing (data/function) members of a class:
There are three methods 1. Using object of a class 2. Using pointer to an object  3. Using Pointer to members itself.

1. Using object of a class:
    student S;
    S.m_public = 500;
    S.showData();

2. Using pointer to an object:
    student S;
    student *sp, *dp; // pointer to a class
    sp = &S;  // pointer to static object
    dp = new student; // pointer to a dynamic object
    sp->m_public = 500; // OR
    *sp.m_public =500;

3. Using Pointer to members itself:
    student S; // static object
    int student:: *p; // pointer to a member data of a class of type int
    p = & student:: m_public; // Accessing the member data
    (or)
    int student:: *p = & student:: m_public;
    int student:: *p = & student:: m_mark; // Error - private not be accessed outside

    int (student:: *fptr) (int, char*); // pointer to a member function of a class
    fptr = & student::setData; // Accessing the member function

    s. *p =425;
    (s. *fptr)(300, "Sanjay");

6. Why are classes with static data members getting linker errors?
Because you cannot initialize static member data using object of a class so it should be initalized explicitly outside the class.

    class Fred {
        public:
        ...
        private:
            int x;
            static int y;   // Declares static data member Fred::y
            (or)
            static int z =10;
    };

    // You must define static data members this way:
    int Fred::y = 42;  // if not defined then you will get compilation error.

8. Use of  (L,U,F) or (l,u,f) suffixes on numeric literals (3.14)?

F or f:
float pi = 3.14f;
By default the float numeric literals(3.14) considered as a double type, unless use 'f' or 'F' suffix to indicate it's type is float.

U or u :
unsigned int i = 786U;
Otherwise 786 will be considered as signed int.

L or l:
long int i = 786L;
Explicitly forcing the compiler that use long arithmetic while using the long int i.

Ex: 
    #define SECONDS_PER_YEAR   (60 * 60 * 24 * 365)UL

This expression wont be suitable for an integer argument on a 16-bit machine-hence the need for the L, telling the compiler to treat the variable as a Long.

9. what is meant by signed char ?
char is an integer type, same as int, short and other integer types. char just happens to be the smallest integer of size always 1 byte. Just like any other integer type, it can be signed or unsigned.

The only main difference between char and other integer types is that plain char is not signed char by default, while with other integer types the signed  by default.

So char can be represented by 3 ways 1. char 2. signed char 3. unsigned char.

By coding standard the plain char is mostly used for character representaion and signed/unsigned char used for 1 byte integers.

10. freopen - Redirects standard input/output:

Using freopen we can read and write the file using standard input and output function/stream (printf,scanf/cin,cout).

    int main() {
        freopen("input.txt","r",stdin);//redirects standard input
        freopen("output.txt""w",stdout);//redirects standard output

        int x;
        cin>>x;  //reads from input.txt
        cout<<x; //writes to output.txt

        scanf("%d",&x);
        printf("%d",x);

        fclose (stdout);
        fclose (stdin);
    }

3. Use Shift Operators:
Integers can be multiplied by any power of two by shifting to the left. And likewise, they can be divided by any power of two by shifting to the right. Multiplication and division by numbers other than powers of two can be accomplished by using sums or differences of shifts. For example, 640 is not a power of two, but 512 and 128 are, so here's the best way to multiply a number by 640 using shifts:
    product=(n<<7) + (n<<9); // n*128 + n*512 = n*640

However, if the processor can multiply in 1–2 cycles, this optimization is worthless.

14. Think about the order of array indices:
Two and higher dimensional arrays are still stored in one dimensional memory. This means array[i][j] and array[i][j+1] are adjacent to each other, whereas array[i][j] and array[i+1][j] may be arbitrarily far apart.
Modern CPUs loads the requested data and adjacent data (a cache line). from main memory into processor cache.This means after array[i][j] is in the CPU cache, array[i][j+1] has a good chance of already being
in cache, whereas array[i+1][j] is likely to still be in main memory.

15. For most classes, use the operators += , -= , *= , and /= , instead of the operators + , - , * , and / .
• For instance: Vector v = Vector(1,0,0) + Vector(0,1,0) + Vector(0,0,1); creates five unnamed, temporary Vectors: Vector(1,0,0), Vector(0,1,0), Vector(0,0,1), Vector(1,0,0) + Vector(0,1,0), and Vector(1,0,0) +Vector(0,1,0) + Vector(0,0,1).
• The slightly more verbose code: Vector v(1,0,0); v+= Vector(0,1,0); v+= Vector(0,0,1); only creates two temporary Vectors: Vector(0,1,0) and Vector(0,0,1). This saves 6 functions calls (3 constructors and 3 destructors).
Note: For basic data types, use the operators + , - , * , and / instead of the operators += , -= , *= , and /= .

16. Jumps/branches are expensive. Minimize their use whenever possible.
• Function calls require two jumps, in addition to stack memory manipulation.
• Prefer iteration over recursion.
• Use inline functions for short functions to eliminate function overhead.
• Move loops inside function calls (e.g., change for(i=0;i<100;i++) DoSomething(); into DoSomething(){ for(i=0;i<100;i++) { ... } } ).
• Long if...else if...else if...else if... chains require lots of jumps for cases near the end of the chain (in addition to testing each condition). If possible, convert to a switch statement, which the compiler some-times optimizes into a table lookup with a single jump. If a switch statement is not possible, put the most common clauses at the beginning of the if chain.
