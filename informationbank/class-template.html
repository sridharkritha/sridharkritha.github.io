Class Template



A. Full Specializations of Class Templates:
    * Specializing the class templates allows you to optimize the implementations of certain types or to fix the misbehaviour of certain types for an
       instantiation of the class template.
    * If you specialize a class template, you must also specialize all its member functions.
    * Although it is possible to specialize a single member function, once you have done so, you can no longer specialize the whole class.
    template<> // Fully specialized
    class Stack<std::string> {
        push(std::string const& elem); 
    };

    void Stack<std::string>::push(std::string const& elem) {
        elems.push_back(elem);
    }

B. Partial Specialization: Class templates can be partially specialized. For example, Consider a class template
    template <typename T1, typename T2>
    class MyClass {
            …
    };

Following are the possible partial specializations:

1. Both template parameters have the same type
    template <typename T>
    class MyClass<T,T> {
    …
    };

2. Second type is int
    template <typename T>
    class MyClass<T,int> {
    …
    };

3. Both template parameters are pointer types
    template <typename T1, typename T2>
    class MyClass<T1*,T2*> {
    …
    };

The following example shows which template is used by which declaration:
    MyClass<int,float> mif;    // uses MyClass<T1,T2>
    MyClass<float,float> mff;  // uses MyClass<T,T>
    MyClass<float,int> mfi;    // uses MyClass<T,int>
    MyClass<int*,float*> mp;   // uses MyClass<T1*,T2*>

If more than one partial specialization matches equally well, the declaration is ambiguous:
    MyClass<int,int> m;        // ERROR: matches MyClass<T,T>
                               //         and MyClass<T,int>
    MyClass<int*,int*> m;      // ERROR: matches MyClass<T,T> 
// and MyClass<T1*,T2*>

To resolve the second ambiguity, you can provide an additional partial specialization for the pointer of the same type:
    template <typename T>
    class MyClass<T*,T*> {
    …
    };

C. Default Template Arguments: For class templates, you can also define default values for the template parameters.
    template <typename T, typename CONT = std::vector<T> >
    class StackDS {
    private:
        CONT elems;               // vector<int> elems; (or) deque<double> elems;
    public:
        void push(T const&);      // push element
    };

    template <typename T, typename CONT>
    void StackDS<T,CONT>::push (T const& elem) {
        elems.push_back(elem);    // append copy of the passed elem
    }

    int main()  {
        // stack of ints:
        StackDS<int> intStack;
        // stack of doubles which uses a std::deque<> to manage the elements
        StackDS<double,std::deque<double> > dblStack;
    }

D. Nontype Template Parameters: Template parameters don't need to be data type. They can also be ordinary values.

    template <typename T, int MAXSIZE>
    class StackDS {
    private:
        T elems[MAXSIZE];        // elements - like dynamic array init
    public:
        StackDS();                 // constructor
        void push(T const&);     // push element
    };

    // constructor
    template <typename T, int MAXSIZE>
    StackDS<T,MAXSIZE>::StackDS() : numElems(0) // start with no elements
    {
        // nothing else to do
    }

    template<typename T, int MAXSIZE>
    void StackDS<T,MAXSIZE>::push (T const& elem) {
        if (numElems == MAXSIZE) {
            cout<<"stack is full";
        }
        elems[numElems] = elem;  // append element
        ++numElems;              // increment number of elements
    }

    int main()  { 
        StackDS<int,20>         intStack;    // stack of up to 20 ints      
        StackDS<std::string,40> stringStack; // stack of up to 40 strings

        // manipulate the stack of up to 20 ints
        int20Stack.push(7);

        // manipulate the stack of up to 40 strings
        stringStack.push("hello");
    }

Again, default values for the template parameters can be specified:
    template <typename T = int, int MAXSIZE = 100>
    class Stack {
    …
    };

E. Restrictions for Nontype Template Parameters: objects with internal linkage are not allowed as nontype template parameters.

Internal linkage Objects: String literals, Floating-point numbers, class-type objects and global pointer - are Not Allowed

External linkage objects: Constant integral values (including enumerations) or pointers to objects - are Allowed

Function Template:
    template <double VAT>        // ERROR: floating-point values are not
    double process(double v)     //        allowed as Nontype template parameters
    {
        return v * VAT;
    }

Class Template:

A: 
    template <std::string name>  // ERROR: class-type objects are not
    class MyClass {              //        allowed as Nontype template parameters
    …
    };

B:
    template <char const* name>
    class MyClass {
    …
    };
    MyClass<"hello"> x;   // ERROR: string literal "hello" is not allowed

C:
    template <char const* name>
    class MyClass {
    …
    };
    char const* s = "hello";
    MyClass<s> x;   // ERROR: s is pointer to object with internal linkage

However, the following is possible:
    template <char const* name>
    class MyClass {
    …
    };
    extern char const s[] = "hello"; // Note: extern
    MyClass<s> x;         // OK
The extern global character array s is initialized by "hello" so that s is an object with external linkage.

F. SubType:
    template <typename T>
    class MyClass {
        typename T::SubType * ptr;
        …
    };
Here, the second typename is used to clarify that SubType is a type defined within a class  T. Thus, ptr is a pointer to the type T::SubType.
    // print elements of an STL container
    template <typename T>
    void printcoll (T const& cont) {
        typename T::iterator it;  // subtype: iterator for the container    
        for (it=cont.begin(); pos!=cont.end(); ++pos) cout << *pos;    
    }

G. Template Template Parameters:
It can be useful to allow a template parameter itself to be a class template. This is useful when a container having another container as an element.

Stack container uses vector container as a element:
    stackDS<int,std::vector<int> > vStack; //integer stack that uses a vector
                // or
    stackDS<int,std::vector> vStack;      // integer stack that uses a vector

Ex:
    #include <allocator>
    template <typename T, 
template <typename ELEM, 
typename = std::allocator<ELEM> > class CONT = std::deque>
    class StackDS {
    private:
        CONT<T> elems;         // elements
    public:
        void push(T const&);   // push element
    };

    template <typename T, template <typename,typename> class CONT>
    void StackDS<T,CONT>::push (T const& elem) {
        elems.push_back(elem);    // append copy of passed elem
    }

    int main()  {
        StackDS<int> intStack;        // stack of ints uses default template 'std::deque' as container
        StackDS<float> floatStack;    // stack of floats uses default template 'std::deque' as container
        
        intStack.push(42); // manipulate int stack        
        floatStack.push(7.7); // manipulate float stack

        // stack for ints using a vector as an internal container
        StackDS<int,std::vector> vStack;
        vStack.push(42);
    }

Note: The second template is a class template so it will be written as
    template <typename ELEM> class CONT = std::deque     // OK
     // not as
    template <typename ELEM> class CONT = std::deque<T>  // Error
    template <typename ELEM> typename CONT = std::deque  // Error

Note: Template template parameters are one of the most recent features required for compilers to conform to the standard. Thus, this program is a good evaluation of the conformity of your compiler regarding template features.

* In template template the keywords struct and union cannot be used:
    template <template<typename X> class C>   // OK
    void f(C<int>* p);
    template <template<typename X> struct C> // ERROR: struct not valid here
    void f(C<int>* p);
    template <template<typename X> union C>  // ERROR: union not valid here
    void f(C<int>* p);

H. Zero/ False Initialization:
There is no default constructor to initializes the fundamental types such as int, double, or pointer types. Using template Zero Initialization we can initialize the fundamental types.
    template <typename T>
    void foo() {
        T x;      // x has undefined value if T is built-in type
    }

by Zero/false Initialization we can write as
    template <typename T>
    void foo() {
        T x = T();   // x is zero (or false) if T is a built-in type
           // x is initialized by calling constructor if T is user defined
    }

Method 2: Initialization list
    template <typename T>
    class MyClass {
    private:
        T x;
    public:
// ensures that x is initialized even for built-in types
        MyClass() : x() { }
    }

I. Linkage of Templates: Every template must have a name and that name must be unique within its scope. That is a class template cannot share a name with a different kind of entity.

Case 1:
    int C;
    class C;  // OK: class and non-class names are in a different "scope"

Case 2:
    int X;
    template <typename T>
    class X;  // ERROR: conflict with variable X

Case 3:
    struct S;
    template <typename T>
    class S;  // ERROR: conflict with struct S

J. ->template and .template Usage:
 In general, a C++ compiler is required to treat a '< ' following the name of a template as the beginning of a template argument list; otherwise, it is a "less than" operator. Some cases the programmer explicitly tells the compiler to consider it as a template by using the keyword "template".

    template<typename T>
    class Shell {
    public:
        template<int N>
        class In { // Inner class
        public:
            template<int M>
            class Deep { // Inner of Inner class
                public:
                virtual void f();
            };
        };
    };
    
    template<typename T, int N>
    class Weird {
    public:
      void case1(Shell<T>::template In<N>::template Deep<N>* p) {
          // p.Deep<N>::f() // Error: will be parsed as ((p.Deep)<N)>f().
          p->template Deep<N>::f();  // inhibit virtual call
      }

     void case2(Shell<T>::template In<T>::template Deep<T>& p) {
            p.template Deep<N>::f();   // inhibit virtual call
     }
  };
