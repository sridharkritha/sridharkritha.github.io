                                    XML


1. The Extensible Markup Language (XML) is a portable, human-readable format for exchanging text or data between programs. XML is derived from the parent Standard General Markup Language (SGML).

2. XML is case-sensitive.

3. XML parser is simply a program or class that reads an XML file.

4.    Some XML parsers :  

                  1. SAX  [ Simple API for XML ]
                  2. TAX [Trivial API for XML]
                  3. DOM [Document Object Model] 
                  4. JDOM [Java Document Object Model]
                  5. JAXP [Java API for XML Processing]   

5.  SAX: Parser converts an XML document into a sequence of Java method calls.
     [SAX-style XML parsing is known as “push parsing” because the parser "pushes" events to your event handler methods.]

6. TAX: It is also possible to parse XML using the reverse architecture, in which your code "pulls" tokens from the XML parser as needed. (Traditional Method)

7. DOM: Apart from reading the XML file, it constructs a graph corresponding to            the elements and their attributes.

8.Parsers checks if the XML file have correct elements by
              1.    Document Type Definition (DTD)
                                       [Or]
              2.    XML Schema (Uses object oriented approach, so we can use inheritance features)


9. In addition to parsing XML, you can use an XML processor to transform XML into some other format, such as HTML. This is a natural for use in a web servlet : if a given web browser client can support XML, just write the data as-is, but if not, transform the data into HTML. We'll look at two approaches to XML transformation:
              1.    Transformation using a generic XSLT processor 

                      and
              2.    Using some parsing APIs suitable for customized operations on XML.
10.    If you need to control how an XML document is formatted, for screen or print, you can use XSL (Extensible Style Language).

11. XSL has two parts:
        1.    Tree transformation (for which XSLT was designed, though it can also be used independently) and
        2.    Formatting (the non-XSLT part is informally known as XSL-FO or XSL Formatting Objects).

12.    XSLT : A scripting language for transforming XML.

13.    XSLT( Extensible Style sheet Language for Transformations) : Allows you a great deal of control over the output format. It can be used to change an XML file from one vocabulary into another, where information is passed from one industry-standard vocabulary to a site that uses another.
