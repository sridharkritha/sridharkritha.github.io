Object files (X.o) and executables (a.out) come in one of several different formats. On most SVr4 implementations the format is called ELF ("Executable and Linking Format"). On other systems, the executable format is COFF (Common Object-File Format). And on BSD UNIX, a.out files have a.out format itself. In this section we will give consideration to object files (binary file about a program).

 

There are five basic memory regions in an object file: Code, Data, BSS, Stack and Heap. Sometimes the Data and BSS segments are collectively referred to as the “Data segment".

 1. The Code/Text segment is where the compiled code of the program itself resides.

 

2. The Data segment contains initialized global and initialized static variables [initalizalizes variables values should not be zero, ie, static int i; would not exist in data segment because by default uninitialized static variable are equated to zero. But static int i=6; would exist in data segment. Similarly the string defined by char s[] = "hello world"; would exist in the data part.

 

3. The BSS (Block Started by Symbol) segment starts at the end of the data segment and contains un-initialized global and un-initialized static variables ie., by default these values are equated to zero. For instance a variable declared static int i; would be contained in the BSS segment.

 

4. The Stack segment is where memory is allocated for automatic variables and local variables within functions. Apart from this the stack stores the "housekeeping" information, when a function call is made. This housekeeping information is known as a stack frame or activation record.

 

Suppose a program begins executing in the function main(), space is allocated on the stack for all variables declared within main(). If main() calls a function, func1(), additional storage is allocated for the variables in func1() at the top of the stack .When func1() returns, storage for its local variables is deallocated.

 

Therefore, the activation record should consist of local variables, argument variables of a function and return address (where to jump back to when the called function is finished).

 

5. The heap area begins at the end of the data segment and grows to larger addresses from there. The Heap area is managed by malloc, realloc, and free, which use the brk and sbrk system calls to adjust its size. The Heap area is shared by all shared libraries and dynamic load modules in a process.

 

Note: code segment is the read only area. Heap grows from bottom to top and stack grows from top to bottom of DS. Error in memory allocation occurs when crosses each other.
