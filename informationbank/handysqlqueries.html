Structured Query Language

SQL is a globally accepted standard language that is used to handle database related activities for creating database schemas, updating/inserting/deleting of data, retrieving data, and managing user roles and databases activities.

SQL is packaged and marketed by several companies. A few popular versions of SQL are Oracle SQL (7,8,9i,10g,11i), Microsoft SQL Server (7,2000,2005,2008), IBM DB2, MySQL, Microsoft Access, Sybase etc.

What is a database?

A database is a repository of data, the collectively comprises of table(s) that are used to hold data, views that fetch data from tables, stored procedures, triggers, functions etc. An SQL server may comprise of multiple databases. The relationships between different tables in a database are often referred to as Database Schema.

1. DDL [Data Definition Language] ----- D CAT   + table
                                  D  -- Drop, C – Create, A –Alter, T – Truncate
2. DML [Data Manipulation Language] ---- US ID
                                  U – Update, S – Select, I – Insert, D – Delete
3. TCL [Transaction Control Language] --- CRS
                                   C – Commit, R – Rollback, S – Save point
4. DCL [Data Control language] – GR
                                    G – Grant, R – Revoke

Note:

1. Oracle     – RDBMS [Relational Database Management System]
2. Oracle 8i – ORDBMS [Object Relational Database Management System]

What's the difference between DELETE TABLE and TRUNCATE TABLE commands?
DELETE TABLE is a logged operation, so the deletion of each row gets logged in the transaction log, which makes it slow. TRUNCATE TABLE
also deletes all the rows in a table, but it won't log the deletion of each row, instead it logs the deallocation of the data pages of the
table, which makes it faster. Of course, TRUNCATE TABLE cannot be rolled back.

Note: Create database and Alter database are also belongs to DDL.

2. DML :

List of predicates used along with DML statements:
WHERE - to check for a condition being met in row.
GROUP BY - used for aggregation, which means that the result that is returned is based on grouping of results based on a column aggregation.
HAVING - makes sure that an SQL SELECT statement should only return rows where aggregate values match conditions
ORDER BY - used to set the sequence of the output in terms of being alphabetical, magnitude of size, order of date. It may accompanied by an 'asc' or 'desc' clause

SELECT * FROM t_students where age > 12 and age < 16;

SELECT Roll_No, SUM(Marks) FROM t_students WHERE Class = 5 GROUP BY Roll_No

SELECT student_id, SUM(Marks) FROM t_students WHERE Admission_Date = '01-Apr-2009' GROUP BY student_id HAVING SUM(Marks) > 500

SELECT fname, lname FROM t_students ORDER BY fname DESC;

Syntax for a SELECT statements covering all the options:
SELECT <attributes> [INTO new_table_]  FROM <tables>              ==>4   
WHERE <condition>                                                                      ==> 1
GROUP BY <group attributes>                                                       ==> 2
HAVING <group condition>                                                      ==> 3
ORDER BY <attributes> [ASC | DESC] ];                                         ==> 5

CONSTRAINTS :[included by -> CREATE (or) ALTER commands]

1.    Domain Integrity Constraint 
       a.    Not Null
       b.    Check
2.    Entity Integrity Constraint
       a.    Unique  [Allows Null value, Disallows duplicates]
       b.    Primary [Disallows Null value, Disallows duplicates]
3.    Referential Integrity Constraint 

1. Domain Integrity Constraint:
 a.    Not Null :

1.    Create table sri (name varchar2(5),salary number(4)   constraint  c1 not null, roll  number(4) constraint  c2 not null);
2.    alter table sri  modify name not null; 
               [ add constraint not possible , only modify]

b.    Check : [ For setting some range]

1.    create table sri (name varchar2(4),salary number(4) constraint c1  check(salary<70000));
2.    alter table sri add constraint c1 check(salary<7000);

2. Entity Integrity Constraint:
a.    Unique :
1.    create table sri(name varchar2(4) constraint c1 unique(name));
2.    alter table sri add constraint c1 unique(name);
3.    alter table sri add constraint c1 unique(name,salary,roll);
               [Composite form, max limit 16 entities]
b.    Primary :

1.    create table sri(name varchar2(4) constraint c1 primary key (name));
2.    alter table sri add constraint c1 primary key(name);

3. Referential Integrity Constraint: 

1. create table dhar(emp number(4) constraint c1 references sri(roll));
         [sri table roll entity should be Prime (or) Unique]
2. alter table dhar add constraint c1 foreign key(emp) references sri(roll);

On Delete Cascade : [Deleting parent data ,automatically delete the child data(dependent) who references the parant]

1. alter table dhar add constraint c1 foreign key(emp) references sri(roll) on delete cascade;

JOIN:

1.    Inner Join / SIMPLE JOIN
        a.    Equi Join
        b.    Non-Equi Join

      a. Left Join

      b. Right Join

3.    SELF JOIN

1.  Inner Join / Simple Join : Join operation which uses comparison operator
  
a. Equi Join: Retrieving the rows from two table by ‘ = ‘ operator

1.    select  name,salary from sri, dhar where roll=emp;
2.    select  s.name,d.name from sri s,dhar d where s.salary=d.salary; // same ‘name’ field in both tables.  

3. SELECT name, salary FROM sri INNER JOIN dhar ON sri.roll=dhar.emp;
Note: Inner joins are the SQL Server default. You can use JOIN instead of INNER JOIN.


b. Non Eqi join: Retrieving the rows from two table by using relational operators(>,<,<=, etc. ) other than ‘ = ‘ operator.

1. select  s.name,d.name from sri s,dhar d where s.salary >= d.salary; // same ‘name’ field in both tables.  


2. OUTER JOIN: [simple join + rows do not matches with other table]

a. Left Join: All rows from the left table are returned. NULL values are returned for the table any time a left table row has no matching row in the right table.

SELECT name, salary FROM sri LEFT JOIN dhar ON row=emp ORDER BY salary;

b. Right Join: Reverse of a left outer join.

c. Full Join: Any time a row has no match in the other table, the select list columns from the other table contains null values.

1. select a.name,b.roll from sri a, dhar b where a.roll=b.roll(+);

3. SELF JOIN:  Joining of table to itself.

1.    select  a.name,b.salary from sri a,sri b where  a.salary>b.roll;

How to run a stored procedure in SQL? How to use the Execute command?

A Stored Procedure is a set of instructions saved for invoking repeated execution. A procedure may be used to handle a set of business logic to be performed. Say we want to insert a record in a table, this type of a scenario is repeatitive and hence a stored procedure may take care of this requirement. Further, the stored procedure may be passed parameters by the calling routine, and so it may be reusable. Below is the syntax to create a procedure:

Whats the difference between a Stored Procedure and a Function in SQL?
Primarily, the main difference between a Stored Procedure and a Function in SQL is that a Function has the ability to return a value to the calling routine, where as a Stored Procedure may or may not return a value, depending on how it is programmed. A Function should have a minimum of one return type. This means that while creating a Function in SQL, it should have a return type. Also, the results of a function may be assigned to a Select statement; this isn't possible with Stored Procs.
