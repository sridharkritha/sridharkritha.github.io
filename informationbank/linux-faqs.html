What shell command would you use to look at the start time of a process?

# ps -eo pid,lstart,cmd

What shell command would you use to search a file for a string?

# find . -name "*.*" | xargs grep [string to search]

In awk what command would you use to return the position of a substring in a string?

# awk 'BEGIN { print index(string, sub-string) }'

In shell, if you had two almost identical text files what command would you use to work out which lines were not the same?

# diff file1 file2

In Unix how do you adjust the priority of a process?

# renice {priority} pid

In C running under Unix what system call function do you use to spawn another process?

int fork(), execl(char *path, char *arg0,...,char *argn, 0), system(char *string) etc.,

In C under Unix what function call would you use to check the I/O status of several file descriptors?

int fstat(int fd, struct stat *buf)

In C under Unix what function call would you use to attach to shared memory?

void *shmat(int shmid, const void *shmaddr, int shmflg);

int shmid; /* shmid of attached segment */

char *shmaddr; /* attach point */

int shmflg; /* flags used on attach */

What shell command would you use to determine what user owned a piece of shared memory?

# Ipcs -m

In Unix shell how would you check who had ownership of an already opened and bound socket or port?

# lsof -nl | egrep "TCP|UDP"


tee file
  The tee command can be used to send standard
  output to the screen and to a file simultaneously.

make | tee build.log
Runs the make command and stores its output to
build.log.

Standard error can be redirected through 2> or 2>>
Example:
cat f1 f2 nofile > newfile 2> errfile
Note: 1 is the descriptor for standard output, so 1> is equivalent to >.

Can redirect both standard output and standard error to the
same file using &> :
cat f1 f2 nofile &> wholefile

top ­-> Displays most important processes, sorted by cpu
percentage
You can change the sorting order by typing
M: Memory usage, P: %CPU, T: Time.

kill ­-9 -­1
Kills all the processes of the current user. ­1: means all
processes.

------------ Recovering from stuck graphics -------------------
If your graphical session is stuck and you can no longer type in your
terminals, don't reboot!
It is very likely that your system is still fine. Try to access a text
console by pressing the [Ctrl][Alt][F1] keys
(or [F2],[F3] for more text consoles)
In the text console, you can try to kill the guilty application.
Once this is done, you can go back to the graphic session by
pressing [Ctrl][Alt][F5] or [Ctrl][Alt][F7] (depending on
your distribution)
If you can't identify the stuck program, you can also kill all your
processes: kill ­-9 ­-1
You are then brought back to the login screen.

Comparing files and directories
diff file1 file2
Reports the differences between 2 files, or nothing if the
files are identical.
diff ­r dir1/ dir2/
Reports all the differences between files with the same
name in the 2 directories.
These differences can be saved in a file using the
redirection, and then later re­applied using the patch
command.
